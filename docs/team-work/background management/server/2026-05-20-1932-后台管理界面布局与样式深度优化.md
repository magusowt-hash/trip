# 2026-05-20-1932 - 后台管理界面布局与样式深度优化

## 改动范围

- `src/app/management/layout.module.css`
- `src/app/management/page.module.css`
- `src/app/management/maps/page.tsx`
- `src/app/management/maps/page.module.css`
- `src/app/management/AdminTable.tsx`
- `src/app/management/AdminTable.module.css`

## 改动目标

本次要解决后台管理界面视觉层次不统一、主壳层工具感弱、通用表格页样式分散且大量依赖内联样式的问题，重点优化后台布局与 CSS 美化表现。

## 改动思路

以后台主壳层为核心统一视觉语言，再把首页、地图管理页和通用表格组件接入同一套卡片、间距、渐变背景和交互态规范，避免只优化单页造成风格割裂。

## 改动内容

- 重做后台主壳层样式，强化侧边栏、顶部栏、内容区的分区关系与卡片化层次。
- 优化后台首页的 hero、统计卡、趋势图卡和模块导航卡，使首页形成更完整的看板节奏。
- 将 `AdminTable` 从内联样式重构为 CSS Module，统一搜索栏、批量操作、表格、分页和弹窗视觉样式。
- 补充新的 `AdminTable.module.css`，作为后台表格类页面的公共视觉基础。
- 调整地图管理页结构，增加摘要信息区和更明确的卡片入口表现。
- 修复后台首页 loading 动画缺少 `@keyframes spin` 的样式遗漏。
