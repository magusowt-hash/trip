# 2026-05-20-2029 - 收敛退出入口并为用户详情接口加兜底

## 改动范围

- `src/app/management/page.tsx`
- `src/app/management/layout.module.css`
- `src/app/management/users/[id]/page.tsx`
- `src/app/api/admin/users/[id]/route.ts`

## 改动目标

本次要解决后台退出入口重复、侧边栏分类头仍偏弱，以及用户详情接口在部分非核心数据查询失败时会直接返回 500 的问题。

## 改动思路

收敛后台交互出口，只保留顶部栏统一退出入口；同时继续强化后台侧边栏分类头层级，并给用户详情接口增加更保守的降级处理，先保证页面可用性。

## 改动内容

- 删除后台看板页里的“退出当前登录”按钮，只保留顶部栏退出入口。
- 放大 `系统管理`、`用户管理` 两个分类头字体。
- 删除用户详情页“最近计划 / 最近好友 / 最近消息”三块下方的说明文案。
- 为用户详情接口中的帖子、评分、收藏、计划、评论、存储文件等查询补充失败降级，避免其中单项异常直接让整页接口 500。
