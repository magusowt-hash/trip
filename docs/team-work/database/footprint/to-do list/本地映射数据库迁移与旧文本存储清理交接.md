# 本地映射数据库迁移与旧文本存储清理交接

## 目标
- 将足迹页“映射本地”的主文件夹记录与资源记录迁移到正式数据库表
- 删除测试期使用的文本存储逻辑
- 让本地映射资源与足迹项直接绑定，不再仅依赖地点名称

## 必须执行的数据库操作
- 执行 migration：
  - [drizzle/0015_local_map_records.sql](/Users/apple/Desktop/codex/trip/drizzle/0015_local_map_records.sql)

## 本轮代码收口
- `/api/footprints/local-map` 已改为数据库读写
- 已删除足迹本地映射测试文本存储逻辑：
  - 旧路径：`test/footprint-local-map/local-map-store.json`
- 前端本地映射请求已改为携带 `footprintItemId`
- 本地映射资源记录已直接绑定到足迹项

## 新表
- `local_map_roots`
  - 主文件夹记录
- `local_map_assets`
  - 主文件夹下的资源记录
  - 与 `footprint_item_id` 直接绑定

## 交接验证项
- 足迹页打开“映射本地”
- 选择主文件夹后可正常读取旧记录
- 调整本地图片位置并保存
- 刷新后重新选择同一主文件夹，旧位置可恢复
- 更换同地点同名但不同足迹项时，不应误串到其他足迹项

## 风险说明
- 在数据库未执行 `0015_local_map_records.sql` 前，新接口无法正常工作
- 旧测试文本文件不会再被新接口读取，若需要迁移历史测试数据，需额外编写一次性导入脚本
