# 前端弹窗最终字段表

## 目标

- 将“挂载网盘”弹窗收口为可直接开发的字段级定义
- 明确弹窗各区块依赖哪些接口字段
- 避免前端在实现时临时拼状态、补语义

## 弹窗定位

弹窗入口：

- 足迹项详情菜单中的 `挂载网盘`

弹窗默认首页：

- `网盘状态总览`

弹窗承载内容：

- 挂载状态
- 连接状态
- 最近同步结果
- 手动同步动作
- 未匹配目录提示

## 弹窗区块

建议固定为 4 个区块：

1. 头部区
2. 状态区
3. 操作区
4. 结果与未匹配区

## 一、头部区字段

用途：

- 告知用户当前操作的是哪个足迹项
- 提供关闭动作

字段表：

| 字段 | 类型 | 必填 | 来源 | 用途 |
|------|------|------|------|------|
| `itemId` | `string` | 是 | `mount/status` | 当前足迹项 ID |
| `itemName` | `string` | 是 | `mount/status` | 当前足迹项名称 |
| `mountState` | `'unmounted' \| 'mounted'` | 是 | `mount/status` | 决定头部状态感知 |

推荐展示：

- 标题：`挂载网盘`
- 副标题：当前足迹项名称

## 二、状态区字段

用途：

- 展示当前是否已挂载、是否可连接、最近同步情况
- 是弹窗默认首页的核心信息

字段表：

| 字段 | 类型 | 必填 | 来源 | 用途 |
|------|------|------|------|------|
| `mountState` | `'unmounted' \| 'mounted'` | 是 | `mount/status` | 决定整体状态分支 |
| `connectionState` | `'unknown' \| 'connected' \| 'disconnected'` | 是 | `mount/status` | 决定颜色和操作按钮 |
| `lastSyncAt` | `string \| null` | 是 | `mount/status` | 展示最近同步时间 |
| `lastSyncStatus` | `'success' \| 'failed' \| null` | 是 | `mount/status` | 展示最近同步结果 |
| `syncState` | `'idle' \| 'syncing' \| 'success' \| 'failed'` | 是 | 前端本地态 + `mount/status` | 控制 loading 表现 |

推荐状态文案：

- `mountState='unmounted'`
  - `未挂载`
- `mountState='mounted' && connectionState='disconnected'`
  - `已挂载，连接异常`
- `mountState='mounted' && connectionState='connected'`
  - `已挂载，连接正常`

推荐颜色：

- 灰：未挂载
- 红：已挂载未连接
- 绿：已挂载已连接

## 三、操作区字段

用途：

- 承载挂载、重试连接、同步、更换、解除挂载等动作

字段表：

| 字段 | 类型 | 必填 | 来源 | 用途 |
|------|------|------|------|------|
| `mountState` | `'unmounted' \| 'mounted'` | 是 | `mount/status` | 决定按钮集合 |
| `connectionState` | `'unknown' \| 'connected' \| 'disconnected'` | 是 | `mount/status` | 决定主按钮 |
| `syncState` | `'idle' \| 'syncing' \| 'success' \| 'failed'` | 是 | 前端本地态 | 控制按钮禁用 |
| `canSync` | `boolean` | 是 | 前端派生 | 是否允许点击同步 |

### 操作按钮规则

#### 1. 未挂载

显示：

- `选择网盘`
- `确认挂载`

#### 2. 已挂载未连接

显示：

- `重试连接`
- `重新选择网盘`

#### 3. 已挂载已连接

显示：

- `同步网盘`
- `更换网盘`
- `解除挂载`

### `canSync` 派生规则

```ts
const canSync =
  mountState === 'mounted' &&
  connectionState === 'connected' &&
  syncState !== 'syncing';
```

## 四、同步结果区字段

用途：

- 在同步完成后明确反馈结果
- 不只 toast，而是在弹窗中保留具体摘要

字段表：

| 字段 | 类型 | 必填 | 来源 | 用途 |
|------|------|------|------|------|
| `lastSyncStatus` | `'success' \| 'failed' \| null` | 是 | `mount/status` | 决定结果区状态 |
| `lastSyncAt` | `string \| null` | 是 | `mount/status` | 展示同步时间 |
| `lastSyncSummary` | `object \| null` | 是 | `mount/status` | 展示计数摘要 |

`lastSyncSummary` 结构：

```ts
type LastSyncSummary = {
  importedAssetCount: number;
  skippedAssetCount: number;
  matchedFolderCount: number;
  unboundFolderCount: number;
} | null;
```

推荐展示字段：

- 导入图片数
- 成功匹配目录数
- 未匹配目录数
- 跳过图片数

### 空状态

当：

- `lastSyncAt = null`

展示：

- `尚未同步`

## 五、未匹配区字段

用途：

- 让用户知道哪些目录未进入正式足迹展示

字段来源建议：

- 单独请求 `GET /api/footprints/cloud/hints`

返回结构：

```ts
type UnboundCloudHintDto = {
  folderId: string;
  folderName: string;
  assetCount: number;
  status: 'unbound';
  reason: 'no_place_match';
};
```

区块字段表：

| 字段 | 类型 | 必填 | 来源 | 用途 |
|------|------|------|------|------|
| `unboundFolderCount` | `number` | 是 | `mount/status` | 常驻数量提示 |
| `unboundAssetCount` | `number` | 是 | `mount/status` | 辅助摘要 |
| `hints` | `UnboundCloudHintDto[]` | 是 | `cloud/hints` | 逐条显示目录 |

### 展示规则

每条显示：

- `folderName`
- `assetCount`

示例：

- `未识别地点A · 11 张`

### 空状态

若 `hints.length === 0`：

- `当前没有未匹配目录`

## 六、挂载选择区字段

这是未挂载状态下才显示的内容。

如果第一阶段需要在弹窗里选择可挂载网盘，建议单独依赖一个候选接口。

推荐结构：

```ts
type CloudMountCandidateDto = {
  rootPath: string;
  displayName: string;
  provider: 'alist';
  connectionState: 'connected' | 'disconnected';
};
```

字段表：

| 字段 | 类型 | 必填 | 来源 | 用途 |
|------|------|------|------|------|
| `candidates` | `CloudMountCandidateDto[]` | 否 | `mount/candidates` | 未挂载时选择网盘 |
| `selectedRootPath` | `string \| null` | 否 | 前端本地态 | 当前选中项 |

说明：

- 若第一阶段并不提供多个候选网盘，而是固定用户空间，也可不单独请求候选列表

## 七、弹窗接口最小组合

为了驱动整个弹窗，前端最少需要：

### 1. 状态接口

`GET /api/footprints/cloud/mount/status?itemId={itemId}`

建议返回：

```ts
type FootprintCloudMountStatusDto = {
  itemId: string;
  itemName: string;
  mountState: 'unmounted' | 'mounted';
  connectionState: 'unknown' | 'connected' | 'disconnected';
  syncState: 'idle' | 'syncing' | 'success' | 'failed';
  unboundFolderCount: number;
  unboundAssetCount: number;
  lastSyncAt: string | null;
  lastSyncStatus: 'success' | 'failed' | null;
  lastSyncSummary: {
    importedAssetCount: number;
    skippedAssetCount: number;
    matchedFolderCount: number;
    unboundFolderCount: number;
  } | null;
};
```

### 2. 未匹配接口

`GET /api/footprints/cloud/hints?itemId={itemId}`

建议返回：

```ts
type FootprintCloudHintsDto = {
  itemId: string;
  totalFolders: number;
  totalAssets: number;
  hints: UnboundCloudHintDto[];
};
```

### 3. 挂载动作接口

`POST /api/footprints/cloud/mount/connect`

请求：

```ts
type ConnectCloudMountRequest = {
  itemId: string;
  rootPath: string;
};
```

### 4. 同步动作接口

`POST /api/footprints/cloud/sync`

请求：

```ts
type SyncCloudRequest = {
  itemId: string;
};
```

## 八、前端本地状态建议

除了接口返回，弹窗本地还应维护：

```ts
type CloudMountModalLocalState = {
  loadingStatus: boolean;
  connecting: boolean;
  syncing: boolean;
  selectedRootPath: string | null;
};
```

说明：

- `syncState` 最终展示可由后端摘要状态和本地 `syncing` 合成

## 九、字段优先级

前端渲染顺序建议：

1. 先拿 `mount/status`
2. 再根据 `unboundFolderCount` 决定是否拉 `cloud/hints`
3. 用户点击同步后，重新拉 `mount/status`
4. 再按需刷新 `cloud/hints`

原因：

- `mount/status` 是弹窗主数据源
- `hints` 只是补充区块

## 十、区块显示条件

### 状态区

- 始终显示

### 操作区

- 始终显示，但按钮集合随状态变化

### 同步结果区

- 始终显示
- 若未同步则显示空状态

### 未匹配区

- 建议始终显示
- 无数据时显示空状态

## 十一、不建议放进弹窗首版的字段

- 单图缩略图墙
- 未匹配目录下所有文件列表
- 自动候选地点列表
- 复杂 provider 元信息
- 原始网盘鉴权信息

原因：

- 第一阶段只做最小可用闭环

## 十二、推荐组件 Props 结构

```ts
type CloudMountModalProps = {
  itemId: string;
  open: boolean;
  onClose: () => void;
};
```

弹窗内部自行请求：

- `mount/status`
- `cloud/hints`

而不是要求父组件预先拼好全部数据。

## 结论

弹窗的最终字段应以 `mount/status` 为主，以 `cloud/hints` 为补充。  
只要这两组数据稳定，前台就能完整渲染：

- 菜单状态
- 状态总览
- 操作按钮
- 同步结果
- 未匹配目录
