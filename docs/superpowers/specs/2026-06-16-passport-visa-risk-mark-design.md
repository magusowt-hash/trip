# 中国护照签证测试前台详情页风险图标设计

## 目标

在测试前台详情抽屉 `test/app/passport-visa/page.tsx` 的标题区，把国家中文名、英文名所在区域右侧空白位置用于展示风险图标。

图标规则：

- `低风险`：绿色盾牌，内部带白色对号
- `中风险`：黄色警示三角
- `高风险`：橙红色警示三角
- `请勿前往`：红色禁止标识

要求：

- 图标使用 SVG 绘制
- 不依赖 emoji 或系统字体
- 放在标题区右侧，不打乱现有抽屉结构
- 现有下方风险 badge 先保留

## 范围

仅修改测试前台详情抽屉：

- `test/app/passport-visa/page.tsx`
- `test/app/passport-visa/page.module.css`
- 新增风险图标 helper / 组件
- 新增与该 helper 直接相关测试

不在本次范围内：

- 正式地图包前端
- 搜索结果卡片
- 地图点位图标
- 后台编辑页
- 风险 badge 文案删除

## 现状

详情抽屉标题区当前结构：

- 左侧国旗
- 中间中文名与英文名
- 右侧关闭按钮

国家风险信息当前只在标题下方 badge 行中以文字形式展示，且仅在非低风险时显示。

标题区右侧在文案块和关闭按钮之间存在可利用空白，但没有风险状态的第一视觉提示。

## 方案

### 结构

在 `drawerTitleRow` 中新增一个固定宽度的风险图标容器：

1. 国旗
2. 标题文本块
3. 风险图标容器

关闭按钮保持在 `drawerHeader` 右侧，不并入该行逻辑。

### 组件边界

新增小型 helper / 组件文件，例如：

- `test/lib/passportVisaRiskMark.tsx`

职责：

- 根据 `riskLevel` 返回 SVG 标记
- 提供稳定的颜色和 aria label 语义

不把整段 SVG 内联塞进 `page.tsx`，避免页面文件继续膨胀。

### 风险映射

- `低风险`
  - 图形：盾牌 + 对号
  - 主色：绿色
- `中风险`
  - 图形：警示三角 + 感叹号
  - 主色：黄色
- `高风险`
  - 图形：警示三角 + 感叹号
  - 主色：橙红
- `请勿前往`
  - 图形：禁止圆环斜杠
  - 主色：红色

### 样式

新增两个样式类：

- `riskMarkWrap`
- `riskMark`

其中：

- `riskMarkWrap` 固定宽度，右对齐
- `riskMark` 统一尺寸约 `44px`

这样中文名和英文名区域不会因不同风险等级而横向跳动。

## 测试

先写失败测试，再实现。

测试覆盖：

1. `riskLevel -> tone` 映射正确
2. `低风险` 返回 `shield-check` 语义
3. `中风险/高风险` 返回 `warning` 语义，但颜色不同
4. `请勿前往` 返回 `prohibited` 语义

本次不做 DOM 级快照测试，先锁定纯函数输出，降低维护成本。

## 结论

本次采用“小型风险图标 helper + 标题区右侧固定容器”的方案，以最小侵入方式把风险状态前置到详情抽屉的第一视觉层级。
