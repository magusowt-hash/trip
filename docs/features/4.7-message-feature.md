## 错误总结

### 1. TypeScript类型命名冲突
**问题**：
- `services/api.ts` 导出了 `ChatMessage` 接口
- `MessagesClient.tsx` 内部定义了同名的 `ChatMessage` 类型
- 导致多处类型冲突，编译失败

**解决**：
- 多次重命名本地类型：`LocalMessage` → `MessageData` → 最终统一使用
- 最终方案：在api中导出 `ChatMessage`，组件内使用 `MessageData`

**教训**：
- 本地类型和导入类型避免同名
- 统一类型命名规范

### 2. JWT认证中间件401错误
**问题**：
- 前端发送请求时携带 `Authorization: Bearer token`
- 后端中间件读取 `req.headers.authorization` 返回 `undefined`

**原因**：
- 中间件中使用了错误的header读取方式

**解决**：
```typescript
// 修改前
const authHeader = req.headers?.authorization;

// 修改后  
const authHeader = req.headers?.authorization ?? req.headers?.Authorization;
```

### 3. 后端中间件路径匹配问题
**问题**：
- 设置了全局前缀 `api`，中间件中检查路径 `/user/` 但实际路径是 `/api/user/`

**解决**：
```typescript
const path = req.path.replace(/^\/api/, '') || req.path;
```

### 4. 前端WebSocket未生效
**问题**：
- 编写了WebSocket代码但未正确集成到组件

**原因**：
- 之前多次重构导致部分代码未更新
- socket初始化和使用逻辑分散
