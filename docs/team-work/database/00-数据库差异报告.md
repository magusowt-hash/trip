# 数据库与 Schema 差异报告

生成时间：2026-05-20
检查方式：对比 `src/db/schema*.ts` 与 `SHOW TABLES` / `DESCRIBE` 结果

## 一、数据库缺失的表

| 表名 | Schema 文件 | 说明 |
|------|-------------|------|
| comment_likes | schema.ts | 评论点赞表，schema 有定义 |
| embed_access_logs | schema.ts | 嵌入访问日志表，schema 有定义 |
| friendships | schema.ts | 好友关系表（DB 用的是 `friends` 表）|

## 二、数据库多余的表

| 表名 | 说明 |
|------|------|
| friends | 遗留表，应改用 `friendships` |
| messages | 完全未在 schema 中定义 |
| rating_aggregates | 完全未在 schema 中定义 |
| user | 可能为遗留表（已有 `users`） |

## 三、表名不一致

| DB 表名 | Schema 表名 | 说明 |
|---------|-------------|------|
| friends | friendships | **已修复** — API 代码仍查询 `friends` 表，需同步改为 `friendships` |

## 四、字段差异

### users
- DB：`visited_places` (json)，Schema 无此字段
- DB：`avatar` 是 `longtext`，Schema 是 `text`
- DB：`status` 允许 NULL，Schema 是 `NOT NULL default 'normal'`

### posts
- DB：`content` 是 `longtext`，Schema 是 `text`
- DB：`privacy` 是 `enum('public','private')`，Schema 是 `varchar(16)`
- DB：`topic` 是 `NOT NULL`，Schema 是 `nullable`

### post_images
- DB：`id` 是 `bigint`，Schema 是 `serial (int)`

### comments / favorites / plans / admin_keys / uploaded_files / transport_items / footprints_* / cloud_* / alist_config / packing_* / station_overrides
- DB：时间字段用 `timestamp`，Schema 用 `datetime(6)`

### markers
- DB：`coverImage`，Schema：`cover_image`

### map_pois / user_map_favorites / user_map_footprints / ratings / list_images / cloud_mounts / rail_map_settings / station_overrides
- DB：`id` 是 `bigint unsigned`，Schema 是 `serial (int)`

### lists
- DB：无 `intro` 列
- DB：`cover_image` 是 `varchar(500)`，Schema 是 `text`

### list_images
- DB：`id` 是 `bigint`
- DB：`thumbnailUrl`，Schema：`thumbnail_url`

### list_items
- DB 额外字段：`transport_plane`、`transport_train`、`transport_bus`、`rating_type`、`custom_rating`
- DB 无 `status` 列（Schema 有）

### packing_categories / packing_templates
- DB：`name` 是 `varchar(100)`，Schema 是 `varchar(64)`

### rail_map_settings
- DB：聚类半径值不同
  - DB: `cluster_r_z1=40, z2=28, z3=18, z4=10, z5=8, z6=4`
  - Schema: `cluster_r_z1=44, z2=32, z3=22, z4=14, z5=8, z6=4`
- DB：`dedup` 值不同（DB 偏小）

### station_overrides
- DB：`level_override` / `display_level` 用的是 `'hub','major','local_major','local','deleted'`
- Schema：`'CH','RK','GI','AS','MT','deleted'`

### cloud_mounts
- DB：仅有 `user_id UNIQUE`，缺 `user_id, footprint_item_id` 复合索引

### storage_files
- DB：`id` 是 `int`，Schema 是 `serial (bigint)`
- DB：无 `updatedAt` 字段

### user_footprint_settings
- DB 包含：`show_poi_labels`、`poi_label_color`、`marker_color`、`marker_shape`
- Schema 缺少以上 4 个字段

## 五、API 层 Bug（已知）

`src/app/api/admin/users/[id]/route.ts:174` 查询 `FROM friends`，应改为 `FROM friendships`。

## 六、修复优先级建议

| 优先级 | 内容 |
|--------|------|
| 🔴 紧急 | `friends` 表名 → `friendships`（导致 500 错误） |
| 🔴 紧急 | `comment_likes` 表缺失（功能不可用） |
| 🟡 高 | `messages` 表定义（未在 schema 中） |
| 🟡 高 | `user_footprint_settings` 缺 4 个字段（前端设置失效） |
| 🟡 高 | `station_overrides` enum 值不匹配（级别映射错误） |
| 🟢 中 | 补齐 `embed_access_logs` 表 |
| 🟢 低 | `rating_aggregates`、`user` 表清理 |
| 🟢 低 | `visited_places` 等字段命名规范化 |
