# 评论系统交互增强设计

## 背景
现有的评论组件（位于 `src/modules/post/PostDetailModal/`）已具备基本功能：树形评论结构、回复、点赞、评论发布。为匹配抖音等主流社交应用的评论区交互体验，需要进行全面优化。

## 目标
在现有代码基础上增量增强，实现以下抖音核心体验：
1. 流畅的打开/关闭动画与手势操作
2. 分页滚动加载（下拉刷新、上拉加载、滚动位置记忆）
3. 即时反馈的评论互动（点赞、回复、长按菜单）
4. 完善的输入框体验（字数限制、发送状态、自动聚焦）

## 设计概述
采用**方案一：增量增强现有模态框**，在 `PostDetailModal` 与 `InfoColumn` 组件中直接添加缺失功能，保持代码结构一致，最小化改动范围。

## 详细设计

### 1. 打开/关闭逻辑增强

#### 动画效果
- **Overlay 淡入**：`opacity: 0 → 1`，CSS transition 200ms
- **Frame 上滑**：`transform: translateY(20px) → 0`，CSS transition 200ms
- **样式位置**：`src/modules/post/PostDetailModal/styles/modalStyles.ts`

#### 手势关闭
- **监听事件**：在 overlay 上添加 `touchstart`、`touchmove`、`touchend` 监听
- **触发条件**：下滑位移 >80px 或下滑速度 >0.5 px/ms
- **兼容性**：与现有点击遮罩关闭逻辑共存，`e.target === e.currentTarget`

#### 滚动锁定
- **保持现有**：`document.body.style.overflow = 'hidden'`（已实现）
- **额外处理**：防止评论区域滚动惯性导致的底层页面滚动

#### 状态记忆
- **存储**：关闭时记录 `commentList` 容器的 `scrollTop`
- **恢复**：重新打开时恢复滚动位置
- **作用域**：同一帖子 ID 内有效

### 2. 滚动与加载逻辑

#### 下拉刷新
- **视觉区域**：评论列表顶部添加 -60px 的 pull-to-refresh 区域
- **触发阈值**：下拉 >50px 显示“松开刷新”提示
- **释放操作**：重置页码为 1，重新获取第一页数据，清空现有列表
- **状态反馈**：显示刷新中/刷新完成提示

#### 上拉加载
- **滚动监听**：监听 `commentList` 容器的 `scroll` 事件
- **触发条件**：`scrollTop + clientHeight >= scrollHeight - 50`（距离底部 50px）
- **防重复**：设置 `loadingMore` 状态避免并发请求
- **API 使用**：`/api/posts/${postId}/comments?page=${page}&limit=20`

#### 分页状态管理
```typescript
interface PaginationState {
  page: number;          // 当前页码
  hasMore: boolean;      // 是否有更多数据
  loadingMore: boolean;  // 是否正在加载更多
  scrollPosition: number; // 滚动位置记忆
}
```

#### 加载指示器
- **列表底部**：显示“加载中...”动画
- **无更多数据**：显示“没有更多了”
- **加载失败**：显示重试按钮

### 3. 评论互动逻辑优化

#### 点赞乐观更新
1. **立即反馈**：点击点赞按钮后立即切换图标样式（填充红色）和数字
2. **异步请求**：并发请求 `/api/comments/${id}/like`
3. **失败回滚**：请求失败则恢复原始状态，显示 toast 提示
4. **状态同步**：保持与服务器点赞数的一致性

#### 回复优化
1. **自动填充**：点击“回复”按钮 → 输入框自动填入 `@${author} ` 前缀
2. **输入聚焦**：自动 focus 到输入框，移动端唤起虚拟键盘
3. **取消回复**：提供取消按钮或点击外部取消回复状态
4. **发送清理**：发送成功后清除 `@用户` 前缀和回复状态

#### 长按菜单
- **触发时长**：长按评论 >500ms 显示浮动菜单
- **菜单选项**：【复制评论、删除、举报】
- **交互设计**：点击菜单项执行操作，点击外部关闭菜单
- **权限控制**：仅显示用户有权限的操作（如自己的评论可删除）

#### 删除/举报操作
- **API 端点**：
  - 删除：`DELETE /api/comments/${id}`（需后端支持）
  - 举报：`POST /api/comments/${id}/report`（需后端支持）
- **即时反馈**：成功操作后立即从 `commentList` 中移除对应评论
- **确认提示**：删除操作前显示确认对话框

### 4. 输入框逻辑完善

#### 字数限制
- **最大长度**：500 个字符（含空格）
- **实时显示**：输入框右下角显示 `已输入 ${length}/500`
- **超出处理**：超出时输入框边框变红，禁止发送按钮
- **字符计算**：使用 `string.length` 计算（考虑中文/Emoji）

#### 发送按钮状态
- **空内容禁用**：`content.trim().length === 0` 时按钮 `opacity: 0.5`，`disabled`
- **发送中状态**：显示“发送中...”文本，禁用按钮和输入框
- **发送成功**：恢复初始状态，清空输入框

#### 自动聚焦
- **触发时机**：评论面板打开且 `commentList` 容器渲染完成时
- **移动端考虑**：避免虚拟键盘遮挡重要内容
- **条件控制**：仅在首次打开或特定交互后触发

#### 发送成功处理
1. **清空输入**：`setCommentText('')`
2. **插入列表**：新评论插入 `commentList` 数组开头
3. **滚动到顶**：设置 `commentList` 容器的 `scrollTop = 0`
4. **更新计数**：`localComments + 1`

#### @用户提示
- **Placeholder**：回复时显示“回复 @用户名...”
- **内容前缀**：输入框内容自动以 `@用户名 ` 开头
- **用户编辑**：允许用户修改前缀后的内容
