# 地图POI外部图片组布局算法问题分析与改进方案

从你提供的两张测试效果图和详细规则文档来看，当前算法已经构建了一个非常完整的框架，但在实际执行中出现了**分布不均、连线交叉、局部重叠**等问题。这些问题主要源于**规则执行顺序不合理**、**约束之间互相冲突**以及**局部优化破坏全局平衡**。

## 一、核心问题诊断

### 1\. 边框投影与外法线方向的根本性冲突

- **问题**：当前算法先用外法线计算基础外扩长度，再用边框平衡结果决定最终朝向。这导致**长度计算与实际朝向不匹配**，拐角处的图片组要么被推得过远，要么挤在一起。

- **表现**：第一张图中右上角的图片组 4 明显比其他组离地图更远，而左下角的组 1 和组 8 则过于靠近。

### 2\. 同层优先回填与角度松弛的顺序错误

- **问题**：先扫角回填再做角度松弛，会导致**已经填好的空位被松弛过程重新打乱**，局部组被拉开后形成新的空隙，最终还是需要外扩成多层。

- **表现**：两张图中都存在明显的 \&\#34;一边空一边挤\&\#34; 的情况，说明同层空白区域没有被有效利用。

### 3\. 连线避让的局部性与滞后性

- **问题**：先完成所有落位再做连线避让，此时图片组的位置已经基本固定，只能做微小的角度调整，无法从根本上解决交叉问题。

- **表现**：第二张图中组 2 和组 3 的连线明显交叉，组 5 和组 6 的连线也有交叉趋势。

### 4\. 半径带约束的一刀切问题

- **问题**：虽然对≤9 组的场景跳过了半径带约束，但对于 10\-20 组的中等规模场景，半径带约束仍然会**强行将已经合理分布的组拉回**，导致局部拥挤。

- **表现**：第一张图有 9 个组，理论上应该能单圈完成，但实际还是出现了多层迹象。

## 二、系统性改进方案

### 1\. 重构核心执行顺序（最关键的改进）

将原来的 \&\#34;先长度后方向\&\#34; 改为 \&\#34;先方向后长度\&\#34;，并调整约束的执行顺序：

**原顺序**：

```Plain Text
生成点 → 分层排序 → 外法线长度 → 边框平衡方向 → 扫角回填 → 角度松弛 → 半径约束 → 连线避让
```

**新顺序**：

```Plain Text
生成点 → 分层排序 → 边框平衡方向 → 基于方向计算外扩长度 → 预碰撞检测 → 同层扫角回填 → 全局角度松弛 → 连线优先避让 → 半径带微调
```

**关键变化**：

- 先确定每个点的最终朝向，再基于该朝向计算所需的最小外扩长度

- 在扫角回填之前增加**预碰撞检测**，提前标记高冲突区域

- 将连线避让提前到半径约束之前，优先保证连线不交叉

- 半径带约束改为 \&\#34;微调\&\#34; 模式，只修正极端异常值

### 2\. 边框投影与方向计算的改进

#### 2\.1 边框投影的分层独立化

- **问题**：当前所有层的点都投影到同一个地图边框，导致内层点的方向被外层点干扰。

- **改进**：为每一层生成一个**虚拟边框**，内层点投影到内层虚拟边框，外层点投影到外层虚拟边框。

- **实现**：虚拟边框的大小 = 地图矩形大小 \+ \(层数 × 基础间距\)

#### 2\.2 方向与长度的耦合计算

```javascript
// 改进后的方向与长度计算
function computeDirectionAndLength(point, layerIndex, balancedBorderPos) {
  // 1. 从点指向平衡后边框位置的方向向量

  return { direction, length };
}
```

### 3\. 同层布局算法的重构

#### 3\.1 基于 \&\#34;角度区间\&\#34; 的预分配

- **问题**：当前逐个扫角的方式效率低，且容易出现 \&\#34;先到先得\&\#34; 的不公平情况。

- **改进**：先为每个点预分配一个**基础角度区间**，区间大小与图片组的角半宽成正比。

- **实现**：

    1. 计算同层所有图片组的总角半宽之和

    2. 计算剩余可用角度 = 360° \- 总角半宽

    3. 将剩余角度平均分配给每个点作为 \&\#34;缓冲区间\&\#34;

    4. 每个点的初始角度区间 = \[基础角度 \- 角半宽 \- 缓冲 / 2, 基础角度 \+ 角半宽 \+ 缓冲 / 2\]

#### 3\.2 区间冲突的全局优化

- **问题**：当前角度松弛是局部两两调整，容易陷入局部最优。

- **改进**：使用**贪心区间调度算法**解决区间冲突：

    1. 按区间左端点排序所有点

    2. 遍历每个点，如果与前一个点的区间重叠，则将当前点的区间向右移动到不重叠的位置

    3. 如果移动后超出 360°，则从开头重新开始

    4. 重复 2\-3 直到没有重叠或达到最大迭代次数

#### 3\.3 同层优先回填的正确实现

```javascript
// 改进后的同层回填算法
function fillSameLayer(points, baseRadius) {
  // 1. 按预分配的角度区间排序
  const sortedPoints = sortByAngleInterval(points);
  
  // 2. 初始化已占用的角度区间列表
  const occupiedIntervals = [];
  
  // 3. 逐个放置点
  for (const point of sortedPoints) {
    const { angleInterval, groupSize } = point;
    const [start, end] = angleInterval;
    
    // 4. 在当前半径层寻找第一个可用的连续区间
    const availableInterval = findFirstAvailableInterval(
      occupiedIntervals, 
      groupSize.angleHalfWidth * 2,
      start, 
      end
    );
    
    if (availableInterval) {
      // 5. 如果找到，放置在该区间的中心
      point.angle = (availableInterval.start + availableInterval.end) / 2;
      point.radius = baseRadius;
      occupiedIntervals.push(availableInterval);
    } else {
      // 6. 如果没找到，增加半径并重新搜索
      return fillSameLayer(points, baseRadius + radiusStep);
    }
  }
  
  return points;
}
```

### 4\. 连线避让算法的改进

#### 4\.1 连线交叉的提前预判

- **问题**：当前连线避让是事后修正，效果有限。

- **改进**：在角度区间预分配阶段就**考虑连线交叉的可能性**。

- **实现**：

    1. 对于每一对点，计算它们的连线是否会交叉

    2. 如果会交叉，给这对点的角度区间增加一个 \&\#34;交叉惩罚值\&\#34;

    3. 在区间调度时，优先将有交叉惩罚的点分开

#### 4\.2 分层连线避让

- **问题**：当前全局连线避让是按编号顺序，小编号先锁定，大编号只能避让。

- **改进**：按**层优先级**进行避让：

    1. 外层点的连线优先级高于内层点

    2. 同层点按距离地图中心的远近排序，远的优先级高

    3. 优先级低的点可以在更大的角度范围内调整

### 5\. 半径带约束的优化

#### 5\.1 自适应半径带宽度

- **问题**：当前半径带宽度是固定的，不适应不同规模的场景。

- **改进**：半径带宽度与**同层组数量**和**平均组尺寸**成正比：

    ```javascript
    const medianRadius = computeMedianRadius(points);
    const groupCount = points.length;
    const avgGroupSize = computeAverageGroupSize(points);
    
    const radiusBandWidth = medianRadius * 0.2 + avgGroupSize.diagonal * groupCount * 0.05;
    const minRadius = medianRadius - radiusBandWidth / 2;
    const maxRadius = medianRadius + radiusBandWidth / 2;
    ```

#### 5\.2 半径带的软约束

- **问题**：当前半径带是硬约束，超出就强行拉回，导致重叠。

- **改进**：改为**软约束**，只有当超出带宽超过一定阈值时才进行调整：

    ```javascript
    if (point.radius > maxRadius + threshold) {
      point.radius = maxRadius + threshold;
    } else if (point.radius < minRadius - threshold) {
      point.radius = minRadius - threshold;
    }
    ```
