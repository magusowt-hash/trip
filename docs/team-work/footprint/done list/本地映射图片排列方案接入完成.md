# 本地映射图片排列方案接入完成

## 改动范围
- `src/app/(shell)/test-css/page.tsx`
- `src/app/(shell)/test-css/page.module.css`
- `src/components/LocalMapModal.tsx`
- `src/components/LocalMapModal.module.css`
- `src/app/(shell)/user/footprints/page.tsx`

## 改动目标
完成图片排列方案的测试页验证，并将验证通过的排列能力迁移到正式 footprint 页面，且仅作用于“映射本地”导入链路。

## 改动思路
先在 `test-css` 中完成三种排列方案、参数交互与随机规则验证，再将同一套布局逻辑收敛到本地映射弹窗，避免影响上传图片与其他既有图片来源。正式页只在本地映射图片首次落位时消费布局配置。

## 改动内容
- 在测试页完成三种排列方案原型：
  - 整齐排列
  - 错位排列
  - 随机排列
- 收敛随机排列逻辑：
  - 由“列/行共享随机步长”改为“逐张相邻随机距离”
  - 随机附加距离范围扩大为 `1-100`
- 将排列方案迁入本地映射弹窗，支持：
  - 一级方案切换
  - 错位方向切换
  - 横向 / 竖向距离输入
- 将正式页自动排布逻辑改为按弹窗配置执行。
- 为正式页本地映射自动排布增加地图区域硬边界：
  - 随机排列最小中心距加大，避免重叠
  - 图片组在最接近地图的一侧整体推出地图外
  - 额外留出一张图宽的安全距离，避免视觉内包裹
