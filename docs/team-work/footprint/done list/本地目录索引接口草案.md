# 本地目录索引接口草案

## 目标

为 Footprint 正式页提供“映射本地”所需的轻量接口，不上传原图，不接数据库，只读写服务端文本记录。

## 记录键

- 以用户所选主文件夹名称 `rootName` 作为唯一记录键
- 同名主文件夹命中同一份记录
- 若已有记录，前端需警告用户是否覆盖

## 接口

### `GET /api/footprints/local-map?rootName={rootName}`

用途：

- 读取某个主文件夹名称对应的文本记录
- 返回已知主文件夹名称列表，供弹窗提示

返回：

```json
{
  "record": {
    "rootName": "Japan-Trip-2026",
    "savedAt": "2026-05-21T10:00:00.000Z",
    "assets": [
      {
        "relativePath": "Japan-Trip-2026/东京/img001.jpg",
        "folderName": "东京",
        "name": "img001.jpg",
        "size": 123456,
        "lastModified": 1716200000000,
        "matchedPlaceTitle": "东京",
        "frameX": 120,
        "frameY": -80,
        "missing": false
      }
    ],
    "unmatchedFolders": ["临时照片"]
  },
  "knownRootNames": ["Japan-Trip-2026", "Sichuan-2025"]
}
```

### `POST /api/footprints/local-map`

用途：

- 覆盖保存某个主文件夹名称对应的文本记录
- 用于保存图片位置、未匹配目录、缺失文件清理后的最终状态

请求体：

```json
{
  "rootName": "Japan-Trip-2026",
  "assets": [
    {
      "relativePath": "Japan-Trip-2026/东京/img001.jpg",
      "folderName": "东京",
      "name": "img001.jpg",
      "size": 123456,
      "lastModified": 1716200000000,
      "matchedPlaceTitle": "东京",
      "frameX": 120,
      "frameY": -80,
      "missing": false
    }
  ],
  "unmatchedFolders": ["临时照片"]
}
```

返回：

```json
{
  "ok": true,
  "record": {},
  "knownRootNames": []
}
```

## 文本文件结构

```json
{
  "version": 1,
  "roots": [
    {
      "rootName": "Japan-Trip-2026",
      "savedAt": "2026-05-21T10:00:00.000Z",
      "assets": [],
      "unmatchedFolders": []
    }
  ]
}
```

## 规则

1. `rootName` 必填。
2. `relativePath` 作为单文件幂等键。
3. 仅保存已匹配图片的最终位置。
4. 未匹配目录只保存目录名列表，不进入正式足迹页数据源。
5. 已缺失文件在用户确认保存后，从最终文本记录中移除。

## 第一阶段不做

- 不上传图片文件
- 不接数据库
- 不做用户账号隔离
- 不做多主键复合识别
