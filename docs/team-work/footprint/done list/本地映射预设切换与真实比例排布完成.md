## 改动目标
完成本地映射弹窗“排列方案”到“预设”的交互收敛，并让足迹页图片在真实宽高比下继续参与预设排布，同时补齐上传图片宽高元数据落库存储。

## 改动思路
拆成三段收敛：
- 先收敛映射弹窗交互，让旧记录存在时默认关闭预设，只有手动开启才重排并覆盖旧位置。
- 再收敛前端图片显示与排布，让图片按真实宽高比显示，同时把组内间距、组间避让、地图周边找位统一到“无形边框”规则。
- 最后补齐上传图片宽高元数据的存储链路，让上传图与本地映射图都能走同一套比例排布逻辑。

## 改动内容
- 将映射弹窗中的“排列方案”改为“预设”：
  - 有旧主文件夹记录时默认灰置
  - 通过“手动开启”按钮启用
  - 启用前弹出警告确认
  - 确认映射后可用新预设替代原有位置记录
- 调整本地映射应用逻辑：
  - 预设关闭时保留旧图位置
  - 新增且无旧坐标的图片仍自动补位
  - 预设开启时清空旧坐标并整体重排
- 图片渲染层改为：
  - 无边框
  - 无圆角
  - 按原始宽高比显示
- 自动排布逻辑改为基于图片组无形边框找位：
  - 各组不可相互覆盖
  - 新组优先生成在距离地图最近的可用区域
  - 若地图四周已围满，则自动向外围扩圈
  - 已有组新增图片时沿原组方向继续展开
- 修复拖动画布时 `panStartRef.current.tx` 可能为空导致的前端报错。
- 本地映射目录扫描时读取图片真实宽高。
- 上传图片链路新增宽高元数据读取与返回：
  - 使用 `sharp` 读取原图宽高
  - 新增 `storage_files.pixel_width / pixel_height`
  - 新增迁移 `drizzle/0013_storage_files_pixel_size.sql`
- 同步更新数据库文档与 footprint 状态板，明确：
  - 当前代码已接入真实宽高排布
  - 数据库需执行 0013 迁移后上传图片链路才会完整生效
