## 改动目标
修复普通地图来源地点添加到其他足迹组时报 500 的问题，并保持普通地图页“已去”为用户维度的全局状态。

## 改动思路
问题根因是 `user_map_footprints` 仍使用 `(user_id, poi_id)` 唯一约束，导致同一用户同一 POI 无法进入多个组。将唯一约束收敛为 `(user_id, group_id, poi_id)`，允许跨组复用；同时保留普通地图默认入口只向默认组补写的逻辑，避免主地图页“已去”语义漂移。

## 改动内容
- schema 将 `user_map_footprints` 唯一索引改为 `user_map_footprints_user_group_poi_unique`
- 新增 migration `0014_user_map_footprints_group_scope.sql`
- 普通地图主入口继续按“默认组已有则不重复插入默认组”处理
- 后台足迹详情接口改为所有组都返回普通地图来源地点
- 数据库文档同步更新为“同一 POI 可属于多个组，但同组不可重复”
