# 本地目录映射手测清单

## 目标

验证 `映射本地` 第一阶段实现是否满足：

- 选择主文件夹
- 子文件夹名完全匹配地点名
- 未匹配目录不进入正式足迹页
- 已匹配图片进入足迹页
- 图片位置可保存到文本
- 二次进入后重新选择同名主文件夹可恢复位置
- 缺失文件会警告，保存后删除位置记录

## 前置准备

1. 准备一个主文件夹，例如 `Japan-Trip-2026`
2. 其第一层子文件夹中至少包含：
   - 2 个与足迹地点完全同名的目录
   - 1 个故意不匹配的目录
3. 每个匹配目录中放 2-3 张图片

## 用例 1：首次映射

1. 打开 `/user/footprints`
2. 点击 `映射本地`
3. 选择主文件夹
4. 确认弹窗内：
   - 显示主文件夹名称
   - 已匹配图片数正确
   - 未匹配目录数正确
5. 点击 `确认映射`
6. 确认：
   - 已匹配图片进入足迹页
   - 未匹配目录未进入正式足迹页
   - 右上角若有未匹配目录，出现提示入口

## 用例 2：首次保存位置

1. 映射完成后，拖动 2-3 张图片
2. 点击 `保存修改`
3. 确认保存成功，无报错

## 用例 3：刷新后恢复

1. 刷新页面
2. 确认页面不会自动恢复本地图片
3. 确认页面出现“已有本地映射记录，需重新选择主文件夹后恢复”提示
4. 再次点击 `映射本地`
5. 重新选择同名主文件夹
6. 确认：
   - 图片重新进入足迹页
   - 拖动前保存过的位置被恢复

## 用例 4：未匹配目录

1. 保留一个故意不匹配的子文件夹
2. 执行映射
3. 确认：
   - 弹窗中显示该未匹配目录
   - 正式足迹页不显示该目录下图片

## 用例 5：新增文件

1. 在已匹配目录里新增一张图片
2. 重新打开 `映射本地`
3. 重新选择同名主文件夹
4. 确认弹窗显示新增文件数增加
5. 确认新图进入足迹页

## 用例 6：缺失文件

1. 删除一个之前已经保存过位置的图片文件
2. 重新打开 `映射本地`
3. 重新选择同名主文件夹
4. 确认：
   - 弹窗出现缺失文件警告
   - 缺失文件不再进入正式足迹页
5. 点击 `确认映射`
6. 再点击 `保存修改`
7. 确认本次保存后，该缺失文件的位置记录被删除

## 用例 7：同名主文件夹旧记录覆盖

1. 同一个主文件夹名称再次执行映射
2. 确认弹窗提示已有旧记录
3. 选择取消时，不覆盖旧记录
4. 选择继续时，可覆盖旧记录并进入新映射结果

## 用例 8：入口位置

1. 打开足迹组菜单
2. 确认菜单中存在 `映射本地`
3. 打开任一足迹项菜单
4. 确认菜单中存在 `映射本地`

## 用例 9：地点名拖动带整组移动

1. 完成一次本地映射
2. 在画布上按住某个地点名文字并拖动
3. 确认该地点下全部图片一起移动
4. 确认出现 `保存修改`
5. 确认整组图片不会整体压入地图区域

## 用例 10：新增映射图片落位方向

1. 某地点先已有一组图片
2. 在该地点目录下新增 1-2 张图片
3. 重新执行映射
4. 确认新图片优先落在该地点现有图片组中心点相对地图的反方向
5. 确认不是压到地图侧，也不是随机散开

## 当前阶段限制

1. 以主文件夹名称作为唯一记录键
2. 同名不同真实路径主文件夹会命中同一份记录
3. 刷新后必须重新选择主文件夹，才会恢复本地图片
4. 当前位置记录先走文本，不进数据库
