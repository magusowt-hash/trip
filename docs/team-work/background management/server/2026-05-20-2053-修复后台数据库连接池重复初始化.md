# 2026-05-20-2053 - 修复后台数据库连接池重复初始化

## 改动范围

- `src/db/index.ts`

## 改动目标

本次要解决后台接口在访问数据库时出现 `Too many connections`，进而导致后台多个页面请求 500 的问题。

## 改动思路

将数据库连接池改为进程级全局单例，避免在开发态热更新或模块重复加载时反复创建新的 MySQL pool，持续占满数据库连接。

## 改动内容

- 为 `mysql2` 连接池增加 `globalThis` 级别复用。
- 仅在全局不存在 pool 时创建新的连接池实例。
- 补充 `maxIdle`、`idleTimeout`、`queueLimit`、`enableKeepAlive`、`keepAliveInitialDelay` 配置，降低连接长期占用风险。
