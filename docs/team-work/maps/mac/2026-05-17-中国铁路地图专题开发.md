## 改动说明

### 数据流水线

```
geojson/*.pbf (33省+西藏)
  → osmium merge → 合并 PBF (1.4GB)
  → osmium tags-filter w/railway=rail → 204MB GeoJSON
  → osmium tags-filter n/railway=station → 4.6MB GeoJSON

scripts/process-railways.py:
  ├── WGS-84 → GCJ-02 坐标转换
  ├── 端点匹配合并相邻线段 (4位≈11m)
  ├── Douglas-Peucker (tolerance 0.001≈100m)
  ├── 坐标截断 5 位小数
  ├── 分类: 高速(CTCS-3)/城际(CTCS-2)/普速
  ├── 排除非客运线
  ├── 站点: 中文清洗+12306白名单+地铁排除
  ├── 站点分级: CSV行政编码(枢纽/地级/县级/其他)
  └── 输出: railways(3.2MB) + stations(529KB)
```

### 分类体系

| 级别 | 判定 | 颜色 | zoom |
|------|------|------|------|
| 一级枢纽 | HUBS 64站精确匹配 | 红 | ≥6 |
| 二级地级 | 行政编码 deep=1 | 橙(黑字) | ≥8 |
| 三级县级 | 行政编码 deep=2 | 绿 | ≥9 |
| 四级其他 | 12306白名单其余 | 灰 | ≥10 |

省会定位文字 zoom=3 黑字 10px，zoom≥4 消失。
