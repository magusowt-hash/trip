# 菜单状态与接口字段映射表

## 目标

- 将前台“挂载网盘”菜单状态与后端字段建立一一对应关系
- 避免前端自行拼装复杂状态判断
- 为弹窗状态总览、菜单颜色、同步结果提示提供统一口径

## 适用范围

- 足迹项详情菜单中的 `挂载网盘`
- 挂载网盘弹窗的状态区
- 页面常驻“待匹配”入口

## 前端状态对象建议

```ts
type FootprintCloudMountStatusDto = {
  itemId: string;
  itemName: string;

  mountState: 'unmounted' | 'mounted';
  connectionState: 'unknown' | 'connected' | 'disconnected';
  syncState: 'idle' | 'syncing' | 'success' | 'failed';

  unboundFolderCount: number;
  unboundAssetCount: number;

  lastSyncAt: string | null;
  lastSyncStatus: 'success' | 'failed' | null;
  lastSyncSummary: {
    importedAssetCount: number;
    skippedAssetCount: number;
    matchedFolderCount: number;
    unboundFolderCount: number;
  } | null;
};
```

## 菜单项状态映射

### 1. 未挂载

前端判断：

- `mountState = 'unmounted'`

菜单表现：

- `挂载网盘`
- 默认颜色

弹窗状态：

- `未挂载`

### 2. 已挂载未连接

前端判断：

- `mountState = 'mounted'`
- `connectionState = 'disconnected'`

菜单表现：

- `挂载网盘`
- 红色

弹窗状态：

- `已挂载，连接异常`

### 3. 已挂载已连接

前端判断：

- `mountState = 'mounted'`
- `connectionState = 'connected'`

菜单表现：

- `挂载网盘`
- 绿色

弹窗状态：

- `已挂载，连接正常`

## 页面常驻“待匹配”入口映射

前端显示条件：

- `unboundFolderCount > 0`

推荐文案：

- `待匹配 {unboundFolderCount}`

说明：

- 不需要等同步刚结束时才显示
- 只要后端仍有未匹配目录，就保持显示

## 同步中状态映射

前端判断：

- `syncState = 'syncing'`

UI 表现：

- 弹窗内主按钮 loading
- 禁止重复点击 `同步网盘`
- 状态区可显示 `正在同步`

说明：

- 菜单项本身不建议切成特殊颜色
- 同步中状态只在弹窗内部表达

## 推荐后端聚合字段来源

### `mountState`

来源建议：

- 足迹项与网盘 root 是否已建立绑定关系

判断：

- 有绑定关系：`mounted`
- 无绑定关系：`unmounted`

### `connectionState`

来源建议：

- 最近一次连接检测结果
- 或当前读取 root 配置并调用 provider 的健康检查结果

判断：

- 检测成功：`connected`
- 检测失败：`disconnected`
- 未挂载或未检查：`unknown`

### `syncState`

来源建议：

- 前端本地请求态 + 最近一次同步任务状态

判断建议：

- 用户刚点击且请求未结束：`syncing`
- 最近一次同步成功：`success`
- 最近一次同步失败：`failed`
- 无进行中任务且无近期结果：`idle`

### `unboundFolderCount`

来源建议：

- `cloud_asset_folders.match_status = 'unbound'` 的数量

### `unboundAssetCount`

来源建议：

- 所有 `unbound` 目录下的图片数汇总

### `lastSyncAt`

来源建议：

- `cloud_asset_sync_logs.finished_at` 最新一条

### `lastSyncStatus`

来源建议：

- `cloud_asset_sync_logs.sync_status` 最新一条

### `lastSyncSummary`

来源建议：

- 最新 `cloud_asset_sync_logs` 的计数字段

## 建议状态接口

`GET /api/footprints/cloud/mount/status?itemId={itemId}`

响应建议：

```json
{
  "itemId": "12",
  "itemName": "默认足迹",
  "mountState": "mounted",
  "connectionState": "connected",
  "syncState": "idle",
  "unboundFolderCount": 2,
  "unboundAssetCount": 17,
  "lastSyncAt": "2026-05-20T10:00:03.000Z",
  "lastSyncStatus": "success",
  "lastSyncSummary": {
    "importedAssetCount": 184,
    "skippedAssetCount": 36,
    "matchedFolderCount": 8,
    "unboundFolderCount": 2
  }
}
```

## 弹窗按钮映射

### 未挂载

- 主按钮：`确认挂载`
- 次按钮：`取消`

### 已挂载未连接

- 主按钮：`重试连接`
- 次按钮：`重新选择网盘`

### 已挂载已连接

- 主按钮：`同步网盘`
- 次按钮：`更换网盘`
- 危险按钮：`解除挂载`

## 同步结果面板映射

建议直接复用：

- `lastSyncSummary`
- `lastSyncStatus`
- `lastSyncAt`

说明：

- 同步完成后无需再定义另一套结果对象
- 当前接口状态对象就足以驱动结果面板

## 推荐前端判断顺序

1. 先判断 `mountState`
2. 再判断 `connectionState`
3. 再判断 `syncState`
4. 再决定是否显示 `待匹配`

说明：

- 不要反过来先看同步结果，否则会把未挂载状态误判成异常状态

## 边界情况

### 已挂载，但从未同步

返回建议：

- `mountState = 'mounted'`
- `connectionState = 'connected' | 'disconnected'`
- `lastSyncAt = null`
- `lastSyncStatus = null`
- `lastSyncSummary = null`

### 最近同步失败，但当前连接恢复

返回建议：

- `connectionState = 'connected'`
- `lastSyncStatus = 'failed'`

说明：

- 连接状态和最近同步结果不应互相覆盖

### 存在未匹配，但连接当前失败

返回建议：

- `unboundFolderCount` 仍返回真实值
- 前端仍可显示“待匹配”

说明：

- 未匹配是已知状态，不依赖当前连接是否在线
