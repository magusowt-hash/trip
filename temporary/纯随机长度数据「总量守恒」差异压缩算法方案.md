## 一、首选算法：滑动窗口守恒平滑算法（最适配「连续线段 \+ 快速收敛」场景）

### 核心原理

通过**滑动窗口**在局部线段区间内做「削峰填谷」，在窗口内严格保持长度总和不变，同时平滑相邻线段的长度差异，天然适配连续线段的定位点逻辑，速度极快，单次 O \(n\) 时间复杂度。

### 算法步骤

1. **参数定义**

    - 输入：长度数组 `L = \[l₁, l₂, \.\.\., lₙ\]`（n 个连续线段，由 n\+1 个定位点决定）

    - 窗口大小 `k`（奇数，推荐 3/5/7，k 越大平滑越强）

    - 平滑强度 `α`（0\&lt;α≤1，α=1 为完全平滑，α 越小越保留原始特征）

    - 输出：平滑后长度数组 `L\&\#39;`，满足 `sum\(L\&\#39;\) = sum\(L\)`

2. **窗口遍历与局部守恒平滑**
对每个窗口中心 `i`（从 `\(k\-1\)/2` 到 `n\-\(k\-1\)/2`）：

    - 取窗口内的子数组 `W = \[l\_\{i\-\(k\-1\)/2\}, \.\.\., l\_i, \.\.\., l\_\{i\+\(k\-1\)/2\}\]`

    - 计算窗口内均值 `μ = sum\(W\) / k`（窗口内总和不变，均值固定）

    - 对窗口内每个长度 `w\_j` 做平滑：

        ```Plain Text
        w'_j = w_j + α * (μ - w_j)
        ```

    - 物理意义：比均值大的线段「削峰」，比均值小的线段「填谷」，窗口内总和严格不变

3. **边界处理**

    - 首尾不足窗口长度的部分，自动缩小窗口大小（如首尾用 k=3，中间用 k=5），保证边界线段也能平滑

    - 若需严格全局总和不变，最后做一次全局归一化修正：

        ```Plain Text
        修正系数 = sum(L) / sum(L')
        L_final = L' * 修正系数
        ```

### 优势

✅ 速度极快：单次 O \(n\) 线性时间，可实时处理
✅ 完全适配连续线段：天然平滑相邻线段的跳变，符合定位点的空间连续性
✅ 严格守恒：窗口内 \+ 全局双重守恒，总量零损耗
✅ 可调性强：通过窗口大小 k 和平滑强度 α，自由控制压缩力度
✅ 无异常值：不会产生负长度，物理上完全合理

### 示例

输入：`L = \[1, 3, 2, 5, 4\]`，k=3，α=1

- 窗口 1：\[1,3,2\]，均值 = 2 → 平滑后 \[2,2,2\]

- 窗口 2：\[3,2,5\]，均值 = 10/3 → 平滑后 \[10/3,10/3,10/3\]

## 二、进阶算法：守恒式 L2 方差最小化（全局最优压缩）

### 核心原理

把「降低数据差异」转化为**带约束的二次优化问题**：在「总和不变、连续线段平滑、长度非负」的约束下，最小化数据的方差（即数据与均值的平方差之和），得到全局最优的压缩结果。

### 数学模型

目标函数（最小化）：

```Plain Text
minimize  Σ (l'_i - μ)²  （μ为原始数据均值，固定不变）
约束条件：
1.  Σ l'_i = Σ l_i  （总量守恒）
2.  |l'_i - l'_{i+1}| ≤ δ  （连续线段平滑，δ为相邻线段最大允许差值）
3.  l'_i ≥ 0  （线段长度非负）
```

### 快速求解方法

对于连续线段场景，用**逐次投影梯度法**快速求解，避免复杂的 QP 求解器：

1. 初始化：`l\&\#39;\_i = l\_i`

2. 计算梯度：`∇ = 2\*\(l\&\#39;\_i \- μ\)`（指向方差减小的方向）

3. 沿梯度方向更新：`l\&\#39;\_i = l\&\#39;\_i \- α \* ∇`

4. 投影到约束空间：

    - 第一步：强制满足总量守恒 → 对更新后的数组做全局缩放，保证总和不变

    - 第二步：强制满足连续平滑 → 对相邻线段做滑动窗口平滑，保证差值不超过 δ

    - 第三步：强制非负 → 所有小于 0 的长度设为 0，再重新做一次总量守恒修正

5. 重复迭代 3\-5 次即可收敛，速度接近线性算法

### 优势

✅ 全局最优：在约束下得到方差最小的结果，差异压缩效果最好
✅ 完全可控：可精确控制相邻线段的最大差值，完全适配定位点的连续逻辑
✅ 严格守恒：所有步骤都保证总量不变
✅ 可扩展性强：可加入更多约束（如最大 / 最小长度限制）

## 三、专用算法：基于定位点的「线段守恒重分配」（最贴合业务逻辑）

### 核心原理

既然线段长度由定位点决定，直接**调整定位点的位置**，在「总长度不变」的前提下，让相邻线段的长度差异最小化，从根源上解决问题，完全贴合你的业务场景。

### 算法步骤

1. **定位点参数化**
设原始定位点为 `P = \[p₀, p₁, \.\.\., pₙ\]`，线段长度 `l\_i = \|p\_i \- p\_\{i\-1\}\|`，总长度 `L\_total = Σ l\_i`

2. **目标定义**
调整定位点为 `P\&\#39; = \[p\&\#39;₀, p\&\#39;₁, \.\.\., p\&\#39;ₙ\]`，满足：

    - 总长度不变：`Σ \|p\&\#39;\_i \- p\&\#39;\_\{i\-1\}\| = L\_total`

    - 差异最小化：`Σ \(l\&\#39;\_i \- l\_avg\)²` 最小（`l\_avg = L\_total / n`，即平均长度）

    - 边界固定：首尾定位点 `p\&\#39;₀ = p₀`、`p\&\#39;ₙ = pₙ`（若允许首尾调整可放开）

3. **快速求解**
对于直线 / 折线场景，用**动态规划 \+ 贪心调整**快速实现：

    1. 先计算目标平均长度 `l\_avg = L\_total / n`

    2. 从起点 `p₀` 开始，沿线段方向，每隔 `l\_avg` 放置一个新定位点

    3. 若终点不匹配，做全局缩放：将所有中间点沿首尾连线方向缩放，保证终点不变、总长度不变

    4. 若需保留原始定位点的特征，加入权重：`p\&\#39;\_i = λ \* 目标点 \+ \(1\-λ\) \* 原始点`，λ 为调整强度

### 优势

✅ 完全贴合业务：直接从定位点层面调整，结果可直接用于后续业务
✅ 差异压缩极致：可直接让所有线段长度趋近于平均长度，差异几乎为 0
✅ 严格守恒：总长度、首尾位置完全不变
✅ 速度极快：O \(n\) 线性时间，无复杂计算

## 五、关键注意事项

1. **总量守恒的兜底修正**：所有算法在迭代后，都建议加一步「全局缩放修正」，保证总和 100% 不变：

    ```Plain Text
    修正系数 = 原始总和 / 新数据总和
    最终数据 = 新数据 * 修正系数
    ```

2. **连续线段的边界保护**：首尾线段的平滑要弱于中间线段，避免边界出现不合理的跳变

3. **强度控制**：不要一次把平滑强度拉满，建议从 α=0\.3 开始，逐步增加，避免数据完全失去原始特征

4. **异常值处理**：若原始数据有极端异常值，先做一次「守恒式异常值裁剪」：把极端大值的超出部分，平均分配给小值，再做平滑
