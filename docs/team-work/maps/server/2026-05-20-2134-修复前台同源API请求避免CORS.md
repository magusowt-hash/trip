# 2026-05-20 21:34 修复前台同源 API 请求避免 CORS

## 本次目标

- 修复前台请求 `/api/user/profile` 时被错误拼接到外部 `http://121.5.24.138:3002` 导致的 CORS 报错

## 实际改动

- 调整 `src/services/request.ts`
- 新增 `resolveRequestUrl(path)`，对站内接口路径做同源兜底
- 当请求路径为 `/api/...` 或 `/health` 时，直接使用当前站点同源地址
- 保留其余非站内路径对 `NEXT_PUBLIC_API_BASE_URL` 的兼容

## 结果

- 浏览器不再直接跨域请求 `121.5.24.138:3002/api/user/profile`
- 前台用户资料等站内接口恢复走当前站点的 `/api/*`
- 避免因外部服务未配置 CORS 导致前台功能失败

## 注意

- 该修复依赖前端重新部署
- 如果服务器仍返回业务错误，需要继续排查服务端接口本身
