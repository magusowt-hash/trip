# 2026-05-16 18:12 - 改为仅响应高德POI点击

## 改动范围

- `src/components/PlanMap.tsx`

## 改动说明

根据最新确认，将地图点击选点从“点击任意坐标后调用逆地理接口反查附近 POI”改为“仅响应高德点击事件中自带的 `event.poi`”。

新的行为：

1. 点击高德已有 POI / 热点对象时，直接读取事件中的 POI 信息
2. 将 POI 名称、经纬度、地址等信息直接传给页面层
3. 点击普通空白地图时，不再调用 `/api/maps/selection`，也不再根据坐标反查附近 POI

这样更接近“点击高德已有标注文字/地点对象”的交互，而不是“点击某个坐标”。

## 验证方式

1. 打开 `/maps`
2. 点击高德地图上已有明确地点标记或热点文字
3. 确认地图上弹出地点卡片
4. 点击普通空白区域，确认不再弹出卡片

## 注意事项

- 是否能拿到 `event.poi` 取决于高德前端点击事件本身是否把该对象当作热点/POI 返回
- 如果某些视觉上可见的文字并不通过 `event.poi` 暴露，仍可能点不到，这属于高德前端事件能力边界
