## 改动说明

### 数据流水线

数据源：`geojson/` 目录下 33 个省级 OSM PBF 文件（2026-05-15 导出）

```
osmium merge 33省 → china-merged.osm.pbf (1.4GB)
  → osmium tags-filter w/railway=rail → 204MB GeoJSON
  → osmium tags-filter n/railway=station → 4.6MB GeoJSON

scripts/process-railways.py:
  ├── WGS-84 → GCJ-02 坐标转换（全国偏移 100-700m）
  ├── Douglas-Peucker 精简 tolerance=0.008 (~800m)
  ├── 坐标截断 4 位小数
  ├── 着色分类（高铁 #e53e3e / 干线 #3182ce / 支线 #38a169 / 普速 #718096）
  ├── 排除侧线（道岔/货场，16.7 万条）
  └── 输出 238,112 条线路 + 17,279 个站点
```

### 前端集成

**`page.tsx`：**

- 新增 `RailRoute` / `RailStation` 类型
- china-rail tab 激活时 `useEffect` fetch railways.json + stations.json
- 地图区域：china-rail 模式渲染 `<PlanMap routes={...} />`，线路按分类着色
- 右侧面板：`<RailPanel>` 站点搜索（17k 站，前端内存过滤，取前 20 条）

**由于数据量大（238k routes, 49 万坐标点），未做 zoom 分层。后续可优化：**
- zoom < 6 只显示干线
- zoom >= 8 显示全部

## 验证方式

1. 确保 `public/data/railways.json` 和 `stations.json` 存在
2. `npm run dev` 启动前端
3. 打开 `/maps`，点击「中国铁路」页签
4. 地图上应显示彩色铁路线路（红色高铁、蓝色干线、绿色支线）
5. 缩放浏览，线路应贴合高德底图（GCJ-02 对齐）
6. 右侧搜索框输入站名如「北京」，应返回匹配站点列表

## 注意事项

- railways.json 21MB，首次加载需要 2-4 秒（取决于网络），建议后续开启 gzip
- 数据处理脚本 `scripts/process-railways.py` 依赖 Python 3 标准库，无第三方依赖
- 若要更新铁路数据：从 geofabrik 下载最新中国 PBF → osmium 提取 → 重跑脚本
- 站点目前仅展示搜索列表，未接地图 marker（17k 站点全量渲染性能不足，待 zoom 分层后处理）
- osmium-tool 需通过 `brew install osmium-tool` 安装
