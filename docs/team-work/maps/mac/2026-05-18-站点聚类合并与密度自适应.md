# 2026-05-18 - 站点聚类合并与密度自适应

## 改动范围

- `src/components/RailCanvas.tsx` — 重构站点绘制逻辑（121-231 行）

## 改动说明

**问题**：
1. 相近特等站（同城多枢纽站）各自独立绘制，北京区域 5+ 红点挤作一团，仅靠 20px 网格去重无法有效合并
2. zoom 分层刚性：`hub ≥ 6` / `major ≥ 8` / `local_major ≥ 9` / `local ≥ 10`，低 zoom 完全看不到低等级站，无法根据实际可视密度自适应

**方案**：空间聚类 + 密度自适应

### 1. 相近特等站合并显示

- hub 站做像素距离贪婪聚类，阈值随 zoom 变化：
  - zoom < 6：40px（全国视野，同城站全部合并）
  - zoom 6-8：28px
  - zoom 8-10：18px
  - zoom ≥ 10：10px（城市视野，近距站点仍独立显示）
- 聚类后绘制单个红底白芯 marker，中心取聚类内各站坐标均值
- 名称通过 `HUB_CITY_MAP` 映射为城市名（如北京西/北京南/北京北 → "北京"）
- zoom ≥ 10 时标签追加站数（如「北京(5站)」）

### 2. 不再强制 zoom 分层

- 删除 `if (zoom < N && level === 'xxx') continue` 硬门槛
- 所有等级站点均参与绘制，改用**自适应网格去重**替代等级过滤
- 网格单元大小随 zoom 动态调整：
  - zoom < 6：36px（极稀疏）
  - zoom 6-8：24px
  - zoom 8-10：16px
  - zoom ≥ 10：12px（允许密集显示）
- 效果：zoom=5 时全国可见少量 major/local_major 站（而非之前的 0），zoom=8 时已能看到 local_major

### 3. 绘制流程

```
draw():
  1. 收集视野内所有可见站点 + 像素坐标
  2. hub 站空间聚类 → hubClusters[]
  3. 绘制 hub 聚类 marker（红底白芯 r=5）
  4. 其余站点网格去重绘制（橙 major / 绿 local_major / 灰 local）
  5. hub 聚类名称 + 其余站点名称（共用 nameDrawn 去重）
```

## 验证方式

1. `npm run dev` → 打开 `/maps` → 切到「中国铁路」
2. zoom=4：省会文字 + 少量枢纽城市 marker
3. zoom=5-6：各城市枢纽站合并显示为单个红点，无重叠
4. zoom=7-8：major 站开始大量出现，枢纽聚类可能分离
5. zoom=10+：同一城市不同枢纽站逐渐分离，标签显示站数
6. 右面板搜索站点功能不受影响

## 注意事项

- 接口 `RailCanvasProps` 未变，page.tsx 无需改动
- `stations.json` 数据结构和分级体系未变
- `HUB_CITY_MAP` 保留，新增 zoom≥10 时的站数后缀
- 省会文字逻辑不变（zoom<4 显示）
