# 第一阶段应用层接口草案

## 目标

- 让当前 `OuterFrame footprint` 可以消费云端网盘图片
- 保持云端资源为独立资源域，不先与 `storage_files` 合表
- 将“手动同步 -> 文件夹匹配 -> 绑定地点 -> 页面展示 -> 未匹配提示”这条链路定义清楚

## 已确认前提

- 用户手动触发“同步网盘”
- 只扫描网盘根目录第一层文件夹
- 一级文件夹名直接匹配当前足迹地点列表
- 文件夹匹配成功后，该文件夹下所有图片自动归入该地点
- 未匹配目录下图片先入库，但状态为未绑定
- 未绑定资源不进入正式 Footprint，仅做提示
- 用户端和管理端都要有提示入口

## 应用层结构

建议拆成 4 个应用服务：

- `CloudRootSyncService`
- `FootprintFolderMatchService`
- `FootprintCloudViewService`
- `FootprintCloudHintService`

说明：

- 同步、匹配、展示、提示四段职责分开
- 后续多形态 Footprint 扩展时，优先复用同步和匹配层

## 服务职责

### 1. `CloudRootSyncService`

职责：

- 扫描当前用户网盘根目录第一层文件夹
- 读取文件夹下图片资源
- 将目录和图片资源写入云端资源索引域

建议方法：

```ts
type SyncCloudRootParams = {
  userId: number;
  triggeredBy: 'user' | 'admin';
};

type SyncCloudRootResult = {
  rootId: string;
  scannedFolderCount: number;
  importedAssetCount: number;
  skippedAssetCount: number;
  unboundFolderCount: number;
  matchedFolderCount: number;
  startedAt: string;
  finishedAt: string;
};

interface CloudRootSyncService {
  syncCloudRoot(params: SyncCloudRootParams): Promise<SyncCloudRootResult>;
}
```

### 2. `FootprintFolderMatchService`

职责：

- 使用一级文件夹名匹配当前用户足迹地点
- 建立“目录 -> 地点”绑定
- 将该目录下全部图片标记为已绑定到该地点

建议方法：

```ts
type MatchFolderParams = {
  userId: number;
  rootId: string;
};

type MatchFolderResult = {
  matchedFolders: Array<{
    folderId: string;
    folderName: string;
    placeTitle: string;
    assetCount: number;
  }>;
  unboundFolders: Array<{
    folderId: string;
    folderName: string;
    assetCount: number;
    reason: 'no_place_match';
  }>;
};

interface FootprintFolderMatchService {
  matchFoldersToFootprints(params: MatchFolderParams): Promise<MatchFolderResult>;
}
```

说明：

- 第一阶段只支持“完全按目录名命中”
- 暂不引入递归规则、模糊自动绑定、多候选自动决策

### 3. `FootprintCloudViewService`

职责：

- 为当前 `OuterFrame footprint` 提供云端图片视图数据
- 仅返回已绑定资源
- 返回结构对齐统一 DTO / projection 设计

建议方法：

```ts
type GetOuterFrameCloudItemsParams = {
  userId: number;
};

type OuterFrameCloudPlaceDto = {
  placeTitle: string;
  groupId: string | null;
  groupName: string | null;
  assets: Array<{
    assetId: string;
    fileName: string;
    thumbnailUrl: string | null;
    originalUrl: string | null;
    width: number | null;
    height: number | null;
    size: number | null;
    capturedAt: string | null;
    accessState: 'ok' | 'expired' | 'forbidden' | 'missing';
    sourceType: 'cloud';
    projection: {
      x: number | null;
      y: number | null;
    } | null;
  }>;
};

type GetOuterFrameCloudItemsResult = {
  userId: number;
  viewType: 'outer_frame';
  places: OuterFrameCloudPlaceDto[];
};

interface FootprintCloudViewService {
  getOuterFrameCloudItems(
    params: GetOuterFrameCloudItemsParams,
  ): Promise<GetOuterFrameCloudItemsResult>;
}
```

### 4. `FootprintCloudHintService`

职责：

- 为用户端和管理端提供未匹配提示
- 不暴露正式展示数据，只返回提醒所需信息

建议方法：

```ts
type GetUnboundCloudHintsParams = {
  userId: number;
};

type UnboundCloudHintDto = {
  folderId: string;
  folderName: string;
  assetCount: number;
  sampleThumbnailUrl: string | null;
  status: 'unbound';
  reason: 'no_place_match';
};

type GetUnboundCloudHintsResult = {
  userId: number;
  totalFolders: number;
  totalAssets: number;
  hints: UnboundCloudHintDto[];
};

interface FootprintCloudHintService {
  getUnboundCloudHints(
    params: GetUnboundCloudHintsParams,
  ): Promise<GetUnboundCloudHintsResult>;
}
```

## 推荐流程

### 用户端流程

```text
用户点击“同步网盘”
  -> CloudRootSyncService.syncCloudRoot()
  -> FootprintFolderMatchService.matchFoldersToFootprints()
  -> 返回同步摘要
  -> 页面刷新云端已绑定数据
  -> 页面刷新未匹配提示数据
```

### 管理端流程

```text
管理员进入管理页
  -> 查看用户未匹配目录统计
  -> 必要时触发指定用户同步
  -> 查看未匹配目录名称、图片数量、样例图
```

## API 草案

### 1. 用户触发同步

`POST /api/footprints/cloud/sync`

请求：

```json
{}
```

响应：

```json
{
  "ok": true,
  "summary": {
    "rootId": "root_xxx",
    "scannedFolderCount": 12,
    "importedAssetCount": 184,
    "skippedAssetCount": 36,
    "matchedFolderCount": 8,
    "unboundFolderCount": 4,
    "startedAt": "2026-05-20T10:00:00.000Z",
    "finishedAt": "2026-05-20T10:00:03.000Z"

```

说明：

- 当前用户上下文由服务端登录态提供
- 不允许前端自带任意 `userId`

`GET /api/admin/footprints/cloud/hints?userId=123`

- 管理端接口允许查指定用户
- 权限校验沿用现有 admin 鉴权

### 5. 管理端触发某用户同步

`POST /api/admin/footprints/cloud/sync`

请求：

```json
{
  "userId": 123
}
```

说明：

- 仅管理员可用
- 行为仍然是“手动触发”，不变成自动任务

## 状态定义

### 目录级状态

- `matched`
  - 目录名成功命中当前足迹地点
- `unbound`
  - 目录已扫描，但未命中任何地点

### 资源级状态

- `ready`
  - 已入库且文件可访问
- `missing`
  - 远端文件不存在
- `forbidden`
  - 当前配置或授权不可访问
- `expired`
  - 临时访问链接失效，需刷新

### 绑定级状态

- `bound`
  - 资源已归属某个地点
- `unbound`
  - 资源已入库但未归属地点

## 页面消费建议

### 用户端

- Footprint 页面增加“同步网盘”入口
- 成功后显示同步摘要：
  - 新导入图片数
  - 成功匹配目录数
  - 未匹配目录数
- 若有未匹配目录，仅做提示：
  - “发现 2 个未匹配目录，未进入足迹展示”

### 管理端

- 提供用户维度的未匹配目录总览
- 可查看：
  - 用户 ID
  - 目录名
  - 图片数量
  - 样例缩略图
- 第一阶段先不要求管理端直接批量修复绑定

## Projection 写入口径

第一阶段虽然先接当前 `OuterFrame`，但仍建议：

- 云端图片位置写入 projection 域
- 不直接把 `frame_x/frame_y` 写进资源表

建议接口：

`PATCH /api/footprints/cloud/outer-frame/projection`

请求：

```json
{
  "assetId": "asset_1",
  "x": 120,
  "y": -80
}
```

说明：

- 为后续多形态扩展保留边界
- 当前页面不需要知道底层是 local 还是 cloud

## 错误处理建议

### 同步失败

返回结构建议：

```json
{
  "ok": false,
  "error": {
    "code": "CLOUD_SYNC_FAILED",
    "message": "网盘同步失败，请稍后重试"
  }
}
```

### 常见错误码

- `CLOUD_NOT_ENABLED`
- `CLOUD_AUTH_INVALID`
- `CLOUD_ROOT_NOT_FOUND`
- `CLOUD_SYNC_FAILED`
- `CLOUD_ASSET_ACCESS_FORBIDDEN`
- `FOOTPRINT_PLACE_LIST_EMPTY`

## 第一阶段不做的内容

- 自动定时同步
- 递归扫描子目录
- 模糊自动绑定地点
- 全站标准地点库候选
- 用户在页面上直接修正未匹配目录绑定
- 本地上传与云端资源的底层统一建模

## 推荐落地顺序

1. 先定义云端资源表与目录表
2. 实现用户手动同步接口
3. 实现目录名命中足迹地点的匹配逻辑
4. 实现用户端已绑定云端数据查询
5. 实现用户端/管理端未匹配提示接口
6. 最后再接 projection 写入

## 下一步可继续细化

- 云端目录表和资源表字段清单
- 匹配逻辑的大小写、空格、全半角规范
- `OuterFrame` 页面对本地与云端混合展示的最终返回格式
