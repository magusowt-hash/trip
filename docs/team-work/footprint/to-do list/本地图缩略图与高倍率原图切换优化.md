# 本地图缩略图与高倍率原图切换优化

## 目标

在不改变当前足迹页视觉效果与排布规则的前提下，解决本地图大量图片同时显示时的卡顿问题，并支持：
- 总览状态优先显示缩略图
- 高倍率放大时无感切换回原图
- 保持当前拖拽、预设排布、组间避让与相册查看逻辑不变

## 当前问题

- 本地图当前仍可能同时解码与绘制大量原图，导致足迹页卡顿明显。
- 即使图片已改为按真实比例排布，性能瓶颈仍主要来自：
  - 大量图片解码
  - canvas 重绘
  - 高频缩放 / 拖拽时图源切换成本
- 直接一次性为全部本地图生成缩略图，会把卡顿从“显示阶段”转移到“生成阶段”。

## 收敛原则

- 不改变当前用户侧视觉设计：
  - 图片仍保持无边框、无圆角、真实比例
  - 预设排布与组间无形边框规则保持不变
- 本地图缩略图优先走前端会话内存，不引入额外服务端文件持久化。
- 原图仍保留为大图查看与高倍率显示源。
- 优先做低风险、局部收敛，不先大改外框整体架构。

## 待办拆分

### 1. 本地图缩略图生成队列

- 为本地图图片增加 `thumbnailUrl`
- 缩略图在前端内存生成，不落盘
- 采用低并发队列，避免一次性全量生成
- 默认并发建议控制在 `2-3`
- 缩略图最长边默认建议控制在 `320px`
- 生成失败时允许回退原图，不阻断当前映射链路

### 2. 图源分级显示

- 低倍率总览时优先显示缩略图
- 接近高倍率阈值时，预加载当前视口内原图
- 达到阈值后切回原图
- 切换过程不应影响当前布局坐标与拖拽结果
- 当前建议阈值：
  - `scale < 4`：显示缩略图
  - `3.5 <= scale < 4`：预加载当前视口内原图
  - `scale >= 4`：优先显示原图
- 图源切换只影响显示源，不改变排布坐标、命中区域与保存逻辑

### 3. 可见区域优先

- 优先处理当前视口内图片
- 视口外图片延后生成缩略图或延后切原图
- 避免为当前不可见的大量图片提前支付解码成本
- 若当前页已有分组外框，可按以下优先级处理：
  - 当前视口中心附近组
  - 当前可见组
  - 地图四周最近组
  - 其余不可见组

### 4. 内存清理

- 切组、重映射、关闭页面时释放本地图缩略图 URL
- 避免长时间使用后对象 URL 持续累积

### 5. 外框层后续性能收敛

- 评估将 `OuterFrameCanvas` 从常驻轮询进一步收敛为按变化重绘
- 评估 POI 坐标层从持续 RAF 改为按缩放 / 平移变化触发
- 在不影响当前设计的前提下减少无效重绘
- 若仍存在高图量卡顿，再评估：
  - 图片层 / 交互层分离
  - 只重绘当前拖拽组
  - 视口外图片只保留命中边界，不立即解码

## 实施顺序

### 第一阶段

- 接入本地图缩略图字段
- 接入低并发缩略图队列
- 接入低倍率缩略图 / 高倍率原图基础切换
- 补齐对象 URL 释放

### 第二阶段

- 为当前视口内图片增加原图预加载
- 将缩略图生成队列改为视口优先
- 校正切图时的命中范围与高亮边界

### 第三阶段

- 收敛 `OuterFrameCanvas` 重绘触发方式
- 收敛 POI 坐标层刷新频率
- 评估是否需要进一步拆层渲染

## 当前实现进度

- 已开始：
  - 本地图缩略图字段接入
  - 低并发缩略图生成队列接入
  - 低倍率缩略图 / 高倍率原图切换基础能力接入
  - 切组与重映射时缩略图对象 URL 释放接入
- 未完成：
  - 高倍率阈值附近原图预加载
  - 视口优先调度
  - POI / 外框层进一步去轮询优化
  - 缩略图与原图切换过程的无感收敛

## 验收标准

- 同一批本地图在首次进入足迹页时，相较原图直出方案，主观卡顿明显下降
- 大量本地图混排时，低倍率总览不应因原图同时解码而明显阻塞交互
- 放大到高倍率后，当前视口内图片可稳定切回原图
- 图源切换前后：
  - 位置不跳
  - 间距不变
  - 分组边框不变
  - 拖拽与点击命中不异常
- 切组、重映射、关闭页面后，不应持续累积失控的对象 URL

## 不做项

- 当前阶段不将本地图缩略图持久化到服务端
- 当前阶段不重做足迹页整体图片样式设计
- 当前阶段不改变预设排布规则与组间避让规则

## 验证重点

- 本地图数量较多时，首次进入足迹页是否明显减轻卡顿
- 横图 / 竖图 / 方图混排时，缩略图与原图切换前后布局是否稳定
- 放大到高倍率时，图源切换是否无闪烁、无跳位
- 切组或重新映射后，是否存在对象 URL 未释放导致的内存持续上涨

## 风险说明

- 若一次性处理全部缩略图，仍可能阻塞主线程，因此必须保留队列与并发限制
- 若高倍率切原图没有预加载，切换瞬间可能出现闪烁
- 若只优化图片层，不处理外框持续重绘，极端图片量下仍会有剩余卡顿
