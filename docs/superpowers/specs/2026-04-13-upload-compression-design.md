# 上传压缩功能设计

**日期**: 2026-04-13

## 目标

实现前端 + 后端双重压缩，在省流和保持较好视觉质量之间取得平衡。

## 架构

```
用户选择文件 → 前端压缩 → 上传 → 后端压缩 → 存储
```

## 压缩策略

| 文件类型 | 前端压缩 | 后端压缩 |
|---------|---------|---------|
| JPG/JPEG | 是 (quality: 0.8) | 是 (quality: 85) |
| PNG | 是 | 是 (quality: 85) |
| WebP | 否（已是压缩格式）| 是 (quality: 85) |
| 其他 | 否 | 否 |

## 前端改动

- 安装 `browser-image-compression`
- 在上传前使用该库压缩图片
- 压缩参数：`{ quality: 0.8, maxWidthOrHeight: 2048 }`
- 保持现有的多文件上传逻辑不变

## 后端改动

- 安装 `sharp` 库
- 上传接口增加压缩处理逻辑
- 图片：使用 Sharp 二次优化压缩
- 非图片：直接存储
- 压缩失败：回退到原文件存储，记录日志

## 错误处理

- 压缩失败时回退原文件
- 记录压缩日志便于排查
