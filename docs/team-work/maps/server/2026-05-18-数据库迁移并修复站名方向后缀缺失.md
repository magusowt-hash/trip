## 数据库迁移

### 问题

pull 了 mac 成员的提交 `2026-05-18-2337-Zoom显示精细化`，其中新迁移 `0008_zoom_refinement.sql` 直接执行报错：

```
ERROR 1067: Invalid default value for 'mt_show_zoom'
```

原因：`mt_show_zoom` 默认值 `11.0` 和 `mt_fade_start` 默认值 `10.0` 均超出 `decimal(2,1)` 的范围（最大 9.9），应为 `decimal(3,1)`。

### 修复

将迁移 SQL 和 `src/db/schema.ts` 中两字段的 precision 从 2 改为 3，迁移成功执行。`rail_map_settings` 表新增 16 列，Z1–Z4 默认值按迁移更新为新标准。

## 站名方向后缀缺失

### 问题

页面显示时，独立未聚类的 CH/RK 站点（如"北京南""西安西"）标签丢失末尾方向字符，统一显示为基础城市名（如"北京""西安"）。

### 根因

`RailCanvas.tsx` 中 `HUB_CITY_MAP` 用于两个目的：
1. **聚类归组** — 将"北京""北京南""北京西"视为同城站进行空间合并 ✅
2. **标签命名** — 始终用映射后的城市名渲染标签 ❌

当单站形成独立聚类（count=1）时，标签仍走 `HUB_CITY_MAP` 映射，方向后缀被吞。

### 修复

- 聚类对象新增 `originalName` 字段保存原始站名
- 渲染标签时：`count === 1` → 用原始站名（含方向后缀），`count > 1` → 用城市名（如"北京"）
- CH 和 RK 两级聚类均适用此逻辑

## 验证方式

- 生产环境高 zoom 查看"北京南""西安西"等独立显示的 CH/RK 站标签是否含方向后缀
- 低 zoom 同城多站聚合时标签是否仍为城市名（如"北京"）
