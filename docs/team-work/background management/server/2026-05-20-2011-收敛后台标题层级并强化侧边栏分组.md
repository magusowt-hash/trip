# 2026-05-20-2011 - 收敛后台标题层级并强化侧边栏分组

## 改动范围

- `src/app/management/layout.tsx`
- `src/app/management/layout.module.css`
- `src/app/management/page.tsx`
- `src/app/management/page.module.css`
- `src/app/management/AdminTable.tsx`
- `src/app/management/AdminTable.module.css`
- `src/app/management/maps/page.tsx`
- `src/app/management/maps/page.module.css`
- `src/app/management/users/[id]/page.tsx`
- `src/app/management/users/[id]/page.module.css`

## 改动目标

本次要解决后台多个页面顶部标题卡重复、侧边栏分类识别不够强、导航信息过杂以及看板趋势布局不够紧凑的问题，继续把后台视觉语言收敛到更干净的管理端形态。

## 改动思路

保留后台顶栏作为唯一主标题来源，删除页面内部重复的标题与描述卡；同步把侧边栏收敛为纯文字导航，并增强分类分组本身的层级，让后台界面更统一、更克制。

## 改动内容

- 删除后台看板页、地图管理页、通用表格页顶部重复标题卡。
- 删除多处标题下的说明性描述内容，只保留必要的主信息。
- 将后台首页趋势图区改为双列结构。
- 将后台侧边栏改为纯文字导航，删除每项缩写图标和说明文案。
- 弱化后台壳层顶部栏、面包屑和装饰感，使整体更接近极简后台。
- 强化 `系统管理`、`用户管理` 两个分类项的分组显示，为分类头补充独立容器与更强层级。
- 同步删除用户详情页顶部说明和统计说明文字，避免与顶栏标题重复。
