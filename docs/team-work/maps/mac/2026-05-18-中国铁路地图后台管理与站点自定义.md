### 表二：`station_overrides`（站点自定义覆盖）

| 列 | 类型 | 说明 |
|----|------|------|
| id | int PK auto | — |
| station_name | varchar(64) UNIQUE | 匹配 stations.json 中的站名 |
| display_name | varchar(64) | 覆盖显示名（NULL=不改名） |
| level_override | enum('hub','major','local_major','local','deleted') | 覆盖实际级别 |
| display_level | enum('hub','major','local_major','local') | 显示级别（类提高/降低） |
| created_at | timestamp | — |
| updated_at | timestamp | — |

**四种操作映射：**

| 操作用语 | 字段变更 |
|---------|---------|
| 删除某站 | `level_override = 'deleted'` |
| 改名 | `display_name = '新名'` |
| 升降级 | `level_override = 'major'` 等 |
| 类提高/降低 | `display_level = 'major'`，不改 `level_override`（原级别不变，按 major 样式显示） |

## API 设计

### 公开接口（前端地图加载，无鉴权）

```
GET /api/public/station-overrides
→ [{ station_name, display_name, level_override, display_level }]
→ 仅返回有覆盖记录的站点（非全量），响应 < 10KB
→ 与 stations.json 并行 fetch，不阻塞首屏
```

### 管理接口（需 admin 鉴权）

```
GET    /api/admin/maps/rail/settings     → 读取当前设置（首次自动初始化默认行）
PUT    /api/admin/maps/rail/settings     → 更新设置

GET    /api/admin/station-overrides?q=北京 → 列表（支持搜索）
POST   /api/admin/station-overrides        → 新增/更新覆盖
DELETE /api/admin/station-overrides/:name   → 删除覆盖记录
```

## RailCanvas 改动

新增 props：

```typescript
interface RailSettings { /* 全部显示参数，key 对应 rail_map_settings */ }
interface StationOverride { station_name: string; display_name?: string; level_override?: string; display_level?: string; }

interface RailCanvasProps {
  // ... 现有 props
  settings?: RailSettings;
  overrides?: StationOverride[];
}
```

渲染流程调整（`draw()` 内）：

```
1. 构建 overrideMap: Map<站名, Override>
2. 遍历 stations:
   a. 查 overrideMap
   b. level_override==='deleted' → skip
   c. display_name 存在 → 替换 st.name
   d. display_level 存在 → 用它决定颜色/大小/zoom门槛（替代 st.level）
   e. level_override 存在且非 deleted → 替换 st.level（影响聚类分级）
3. 后续聚类/去重/绘制逻辑不变，但阈值/颜色从 settings prop 取值
   （settings 不存在时 fallback 当前硬编码默认值）
```

## 注意事项

- 首次 GET settings 时若表为空，API 自动 INSERT 默认行并返回
- `station_overrides` 仅存储有变动的记录，全量默认的站点不占行
- 公开接口 `/api/public/*` 响应头设 `Cache-Control: public, max-age=300`，Cloudflare/nginx 可缓存
- 设置表通过 `PUT` 整体替换，管理界面自行组装完整 JSON body
- RailCanvas 不改动 `HUB_CITY_MAP`，该映射仍在前端硬编码
- 性能：overrides 在内存中构建 Map 后 O(1) 查表，不增加渲染开销
