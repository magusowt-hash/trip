# 2026-05-20 21:46 新增数据库连接检测脚本

## 本次目标

- 为 `connect ECONNREFUSED 127.0.0.1:3306` 场景提供一个可直接在服务器执行的排查脚本
- 快速区分端口拒绝、认证失败、数据库名错误、连接数异常等问题

## 实际改动

- 新增 `scripts/check-db-connection.js`
- 在 `package.json` 增加命令 `db:check-connection`

## 脚本能力

- 校验 `DATABASE_URL` 是否存在
- 校验 `DATABASE_URL` 格式是否符合 `mysql://user:password@host:port/database`
- 测试目标 `host:port` 的 TCP 连通性
- 测试 MySQL 登录
- 执行 `SELECT 1`
- 输出当前数据库名
- 输出 `Threads_connected`
- 输出 `max_connections`

## 使用方式

```bash
npm run db:check-connection
```

## 结果

- 可直接判断报错属于“3306 未监听/被拒绝”还是“MySQL 已通但认证/库配置错误”
- 为服务器侧恢复后台接口提供明确排查入口
