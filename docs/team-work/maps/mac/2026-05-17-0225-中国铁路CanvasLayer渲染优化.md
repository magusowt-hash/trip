# 2026-05-17 0225 - 中国铁路 CanvasLayer 渲染优化

## 改动范围

- `src/components/RailCanvas.tsx` — 新增：Canvas 自定义渲染组件
- `src/app/(shell)/maps/page.tsx` — 修改：china-rail 改用 RailCanvas 替代 PlanMap routes

## 改动说明

**问题**：之前 238k 条线路通过 `PlanMap routes` prop 渲染为 238k 个 `AMap.Polyline` Overlay，Canvas 每帧遍历所有 Overlay 导致严重卡顿。

**方案**：新建 `RailCanvas` 组件，用一个 HTML Canvas 叠加在地图容器内，在 Canvas 上一次性绘制所有线路。

- Canvas 通过 `document.createElement` 注入到地图容器的 DOM 内，确保 `lngLatToContainer` 坐标对齐
- 监听 `zoomend` / `moveend` 触发重绘，16ms 防抖
- zoom 分层过滤：< 6 只显示 4+ 坐标点的长线 (~750 条)，< 8 显示 3+ 坐标点 (~3.5k 条)
- 视野裁剪：两端点都不在 bounds 内的线路跳过
- DPR 感知：高清屏 2x/3x 渲染

**page.tsx**：china-rail 模式不再传 routes 给 PlanMap，改为通过 `onMapLoad` 获取 map 实例，传给 RailCanvas。

## 验证方式

1. `npm run dev` → 打开 `/maps` → 切到「中国铁路」
2. 地图应瞬间显示彩色铁路线路，无卡顿
3. 缩放地图，线路自动重绘，zoom 小时只显示干线
4. 拖拽时无延迟

## 注意事项

- Canvas 通过原生 DOM API 注入，不依赖 React 渲染周期
- 若 PlanMap 容器尺寸变化（非 window resize），需增加 ResizeObserver
- 后续可在 RailCanvas 上叠加站点 marker（同样用 Canvas 绘制）
