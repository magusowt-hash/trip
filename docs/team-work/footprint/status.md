  - 从组内移除
  - 添加到其他已去组
- 已删除旧“挂载网盘 / AList”用户端入口、管理端入口、服务层实现和接口实现。
- 当前本地目录映射方案已接入 `/user/footprints`，支持：
  - 目录扫描
  - 已匹配地点导入
  - 未匹配目录提示
  - 缺失文件警告
  - 主文件夹记录恢复
  - 映射弹窗内三种图片排列方案
  - 图片真实宽高参与预设排布
  - 精准匹配 + 单向模糊匹配确认
  - 图片组数量小于 `5` 时按四向极值唯一分区，且该规则已完成定向测试通过
  - 图片组文字锚点与连线地图组端点当前已收敛为共享几何计算
  - 地图禁入区当前已修正为仅按地图本体生效，不再随缩放比例异常放大或缩小，且该项已完成定向测试通过
  - 图片组数量小于 `5` 时继续保留正式页原有分区排布，不并入测试页新引擎
  - 图片组数量大于等于 `5` 时，正式页大组自动排布主流程当前已统一为：
    - 原始角度直线出射
    - 顺序最短半径落位
    - 末端矩形碰撞延长
    - 长度回收
    - 单线角度微调
    - 双线联动微调
    - 邻位最小间隔联动微调
  - 当前大组排布已按 `测试排布算法迁移说明-2026-05-31.md` 切换为测试页最终版算法，不再使用旧闭环排序 / 圆周锚点主流程
  - 当前大组排布的矩形语义已收敛为：
    - 图片组渲染矩形：继续使用正式页当前整体组几何
    - 连线锚点：继续使用正式页共享几何中的 `lineAnchorX / lineAnchorY`
    - 排布碰撞矩形：改按测试页矩形语义参与避碰与收敛
  - 默认最大视图当前已改为按图片组隐式矩形四极值拟合：
    - 使用固定组几何取所有组的最左、最右、最上、最下极值
    - 初始视图整体位置按真实极值中心落位
    - 不再以地图中心作为预设生成后的默认视图基准
  - 当前最小缩放已改为“预设最大视图的拟合结果”，不再保留固定 `20%` 下限
  - 右下角当前已显示实时缩放比例，便于页面级排查默认视图与缩放交互
  - 当前文字 / 端点相对规则：
    - `N / E / W`：端点位于文字上方
    - `S`：端点位于文字下方
- `/test-css` 测试页当前已完成一版随机点闭环排序验证页，且本轮测试通过：
  - 支持 `3` 到 `50` 个点的随机生成与重排
  - 当前点位生成已改为全范围独立均匀随机，不做分区均衡、方向修正或预分层
  - 当前测试页已转为图片组模拟验证页，不再保留早期仅看闭环自交的测试指标
  - 当前测试页主要用于验证：
    - 大于等于 `5` 组时的直线出射稳定性
    - 末端矩形避碰后的线长控制
    - 单线 / 双线 / 邻位联动微调后的局部疏密收敛
    - 线段不交叉、矩形不重叠
- “添加到某组”链路已补齐共享相册确认逻辑，且“否”时不会再误继承源相册图片。
- 足迹组管理中的批量添加/删除已改为批量接口，成功后使用局部状态更新与右上角短提示，不再刷新影响观感。
- 本地映射主文件夹记录与资源记录当前已切换为数据库方案：
  - 主文件夹记录：`local_map_roots`
  - 资源记录：`local_map_assets`
  - 资源记录直接绑定 `footprint_item_id`
- `storage_files` 代码侧已回退为本地上传模型，不再通过当前代码路径承接 cloud 来源图片。
- AList 相关数据库扩展与 migration 历史文件暂未删除，后续通过数据库待办文档统一处理。
