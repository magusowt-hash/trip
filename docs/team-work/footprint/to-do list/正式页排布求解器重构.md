# 正式页排布求解器重构

> 已完成：正式页排布主链路已收敛为独立 solver / refiner / constraints / scoring / types 模块，页面侧已改为以 solver 入口为主，不再直接持有主求解编排。后续区域效果优化转入状态板“当前下一步”继续推进。

## 实现内容

- 将 `/user/footprints` 正式页当前排布逻辑从页面内联大函数重构为独立求解器主链路。
- 正式页排布后续不再继续以“单组收缩 + 邻居推挤 + 局部补丁”的方式叠加修复，而改为“初始径向布局 + 扇区簇求解”的两阶段结构。
- 正式页排布目标统一收敛为：
  - 允许整体紧凑
  - 不允许任意连线交叉
  - 不允许 `photoRect` 与 `photoRect` 重叠
  - 不允许 `labelRect` 与 `photoRect` 重叠
  - 不允许 `labelRect` 与 `labelRect` 重叠
  - 在满足上述硬约束前提下，再尽量缩短线长、提升局部簇协调性
- 正式页当前右上、右下、下中等密集区域不再依赖单点微调，而要按扇区簇联合求解。
- `groupRect` 不再作为“硬非法”判定依据，只保留为拥挤度与紧凑度的软约束参考。
- 当前正式页排布中的 `base anchor` 统一改为“初始径向布局结果”，不再回退到 `logicalX / logicalY` 原始地图方向。
- 页面层只保留：
  - 数据分组
  - 调用求解器
  - 应用排布结果
- 候选生成、硬约束校验、簇级评分、局部重排，统一下沉到独立 solver 模块。

## 实现方式

- 新建正式页布局求解模块，建议拆分为以下文件或同级逻辑层：
  - `footprintLayoutTypes`
  - `footprintLayoutConstraints`
  - `footprintLayoutCandidates`
  - `footprintLayoutScoring`
  - `footprintLayoutSolver`
- 页面当前 `PendingPlaceGroup`、`LogicalRect`、候选 placement 等结构，逐步迁移为 solver 专用类型。
- 求解主流程固定为两段：
  - 第一段：`buildRadialLayout()` 负责生成初始稳定骨架与极角顺序
  - 第二段：按扇区簇执行联合求解，决定每组最终半径层、角度微偏和局部让位关系
- 当前页面中的以下逻辑降级为候选生成器或局部求解器内部能力，不再直接主导结果：
  - `findMinimalFeasibleRadiusOnBaseRay`
  - `findMinimalFeasibleRadiusWithMinorAngleAdjust`
  - `refineFeasibleRadiusInterval`
  - `refineFeasibleRadiusLocally`
  - `tryNeighborNudge`
- `evaluatePlacementCandidate()` 保留为统一硬约束入口，但需继续收敛：
  - `photo/photo`、`label/photo`、`label/label`、交叉线、压地图为硬非法
  - `groupRect` 只参与拥挤评分，不参与硬非法
- 增加簇级候选生成与簇级评分：
  - 窗口内多组同步内收
  - 相邻组小幅左右让位
  - 向低密度扇区迁移
  - 保留对初始极角顺序的稳定约束
- 右上重点协调区域（如苏州 / 杭州 / 上饶 / 黄山）与下方区域（如广州 / 汕头 / 深圳 / 香港）均通过簇级求解统一处理，不再做区域特判补丁。
- 厦门、桂林这类“明显存在更优空区”的组，允许在簇级求解中以更大动作空间向内收或向空区迁移，但仍受硬约束控制。

## 当前问题归并

- 当前正式页排布几乎无明显变化，说明现有“局部补丁式优化”无法真正改变整体格局。
- 当前右上区域协调性差，本质是局部簇问题，不是单点半径问题。
- 当前仍存在地点文本与其他组图片重叠风险，说明真实碰撞约束尚未完全成为唯一布局准则。
- 当前厦门无法自然向内收、桂林无法进入左侧空区，说明候选生成与簇级迁移能力不足。
- 当前 `page.tsx` 同时承担页面状态、几何计算、候选生成、约束判断、评分和求解，职责已经过载，不利于继续维护。

## 执行拆分

- 第一阶段：抽 solver 入口并统一页面调用路径
  - 页面只保留 `solvePendingGroupPlacements()` 入口调用
  - 修正 `base anchor` 来源为初始径向布局结果
- 第二阶段：迁出类型、约束、评分与候选生成
  - 从 `page.tsx` 剥离布局相关纯函数
  - 保证行为不变，只做结构迁移
- 第三阶段：重写扇区簇求解主流程
  - 以簇级联合候选替代当前单组 / 双组主导逻辑
  - 保留现有初始径向布局作为稳定骨架
- 第四阶段：继续收敛硬约束与软约束边界
  - 去除 `groupRect` 硬非法
  - 强化真实碰撞与交叉线硬约束
- 第五阶段：再做区域性效果优化
  - 右上协调
  - 厦门内收
  - 桂林迁移
  - 下中与右下局部簇均匀化
