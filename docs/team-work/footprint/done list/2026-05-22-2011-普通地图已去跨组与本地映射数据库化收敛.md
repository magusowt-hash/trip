## 改动目标
- 收敛普通地图“已去”的跨组语义、读取链路和共享相册语义
- 优化足迹组内批量添加/删除与静默反馈交互
- 收敛本地映射的匹配规则，并将主文件夹记录与资源记录迁移到数据库，删除测试期文本存储

## 改动思路
- 普通地图“已去”继续独立于榜单共享相册语义处理，只保留“地点跨组复用”能力，不再误显示为“共享”
- 足迹组管理操作从逐个请求改为批量接口，并在前端直接更新局部状态，避免刷新破坏观感
- 本地映射先做两层匹配收敛，再把主文件夹记录与本地资源记录正式落库，资源记录直接绑定足迹项

## 改动内容
- 普通地图“已去”语义收敛：
  - 普通地图项不再显示“共享”
  - 普通地图项仍支持相册、上传图片、添加到其他组
  - 普通地图添加到其他组不再传递 `share_photos`
  - 普通地图分组写入返回不再误报 `shared: true`
- 普通地图跨组支持补齐：
  - 所有足迹组都可读取普通地图来源地点，不再只限默认组
  - `view` 模式与后台管理端同步支持读取普通地图来源地点
  - `user_map_footprints` 唯一约束从 `(user_id, poi_id)` 调整为 `(user_id, group_id, poi_id)`
  - 新增 migration `0014_user_map_footprints_group_scope.sql`
- 足迹组管理交互优化：
  - 批量“添加到”与批量“删除”改为批量接口
  - 批量添加时榜单项默认不共享相册，不逐条弹确认
  - 添加/删除成功后不再刷新组列表或当前组内容
  - 改为局部更新当前组内容与组计数
  - 右上角增加 3 秒自动消失的操作提示
- 本地映射匹配规则收敛：
  - 第一层：文件夹名与地点名完全一致的精准匹配，自动通过
  - 第二层：对剩余未匹配目录执行单向模糊匹配，规则为 `地点名.includes(文件夹名)`
  - 模糊匹配结果进入确认列表，用户可逐项取消后再确认映射
- 本地映射存储正式数据库化：
  - 删除足迹本地映射测试期 `test/footprint-local-map/local-map-store.json` 读写逻辑
  - 新增 `local_map_roots` 保存主文件夹记录
  - 新增 `local_map_assets` 保存本地映射资源记录
  - 本地映射资源记录直接绑定 `footprint_item_id`
  - 前端本地映射请求改为传递足迹项 `id + title`
  - 新增 migration `0015_local_map_records.sql`
- 数据库文档同步更新：
  - `03-地图与POI.md`
  - `07-云端同步与存储.md`
  - `README.md`
  - footprint 数据库交接文档补齐

## 当前结果
- 普通地图“已去”已可稳定进入多个足迹组，且用户端不会再把其误判为“共享相册”
- 足迹组内批量添加/删除与单项添加/删除已改为更轻的交互反馈
- 本地映射已支持“精准匹配 + 单向模糊匹配确认”
- 本地映射主文件夹记录与资源记录已切换为数据库方案，测试期文本存储已不再用于足迹页链路
