# 2026-05-16 16:52 - 分离前后端高德Key解决JS API鉴权失败

## 改动范围

| 文件 | 改动 |
|------|------|
| `src/components/PlanMap.tsx` | AMAP_KEY 恢复为 Web端 key |

## 改动说明

浏览器报错 `FlyDataAuthTask error: USERKEY_PLAT_NOMATCH`。

之前将三处 Key 统一替换为 `fbf5d9a8e346f93257eb7c5ab4d32034`，该 Key 开通了 Web 服务但**未开通 Web端(JS API)**，导致前端地图加载后内部鉴权失败。

### 修复：前后端 Key 分离

| 位置 | Key | 类型 |
|------|-----|------|
| `PlanMap.tsx`（前端 JS API） | `64138cb3827187cd053ccbb9eaa18fa2` | Web端(JS API) |
| `search/route.ts`（后端 REST） | `fbf5d9a8e346f93257eb7c5ab4d32034` | Web服务 |
| `selection/route.ts`（后端 REST） | `fbf5d9a8e346f93257eb7c5ab4d32034` | Web服务 |

PlanMap 的地图点击逆地理走的是 `/api/maps/selection`（服务端代理），不直接调用高德 REST API，所以前端只需 JS API Key。

## 验证方式

- 打开 `/maps` 页面，地图应正常加载无报错
- 搜索正常返回结果
- 「地图选点」点击地图能触发逆地理

## 注意事项

- 两个 Key 分别在高德控制台管理，后续变更需确认是改哪个
- 如需新 Key 同时支持 JS API + Web 服务，可在控制台同一个 Key 下开通两种平台
