# 榜单管理 - 批量图片上传设计

## 概述

在后台管理界面的榜单详情页，为榜单项(list_items)提供批量上传封面图的功能。系统根据文件名与数据标题的精确匹配，自动将图片绑定到对应的榜单项，并在覆盖时自动删除旧图文件。

## 目标

- 为榜单详情页新增「批量上传图片」入口
- 支持多图选择和拖拽上传
- 文件名（去扩展名）与 `title` 精确匹配（不区分大小写）
- 覆盖时自动删除旧图片文件

## 数据模型

- **目标表**：`list_items`
- **绑定字段**：`coverImage`
- **匹配字段**：`title`（精确匹配，不区分大小写）

## 功能设计

### 入口

在榜单详情页 `/management/lists/[id]/page.tsx` 内，在现有操作按钮区域新增「批量上传图片」按钮。

### 弹窗 UI

Modal 形式，包含：

1. **文件选择区**：拖拽或点击选择图片，支持 jpg/png/webp/gif，单次可选择多张
2. **匹配预览列表**：
   - 每行：`[缩略图] 文件名 → 标题`  + 状态标签
   - 状态标签：
     - 绿色「待上传」- 新绑定
     - 黄色「将覆盖」- 已有关联图片，将被替换
     - 红色「未匹配」- 文件名无对应标题，默认跳过
   - 未匹配项默认勾选「跳过」，可手动取消勾选尝试强制匹配
3. **底部操作栏**：「取消」+ 「确认上传」按钮，显示待上传数量

### 交互流程

1. 用户选择图片 → 前端读取文件名（去扩展名）
2. 与该榜单下所有 `title` 不区分大小写匹配
3. 显示匹配预览列表
4. 用户确认 → 批量执行：
   - 每张图调用 `POST /api/upload` 上传
   - 若该 item 已有旧图，调用 `DELETE /api/upload` 删除旧文件
   - 每张图调用 `PUT /api/admin/list_items` 更新 `coverImage`
5. 显示结果：成功 N 张 / 失败 M 张 / 跳过 K 张

### 错误处理

- 单张失败不影响其他
- 网络错误自动重试一次
- 未匹配的文件默认跳过
- 上传过程可中断

## API 设计

### 新增：删除图片文件

**DELETE /api/upload**

- 请求：`{ url: string }`（JSON body）
- 功能：根据 URL 删除 `public/uploads/` 下的文件
- 响应：`{ success: true }`
- 错误：文件不存在返回 200，删除失败返回 500

### 修改：更新图片时自动删除旧图

`PUT /api/admin/list_items` 逻辑调整：
- 当 `cover_image` 有新值时，提取该 item 当前的 `coverImage` 旧值
- 异步删除旧文件（不影响主流程）

## 文件清单

1. `src/app/api/upload/route.ts` - 新增 DELETE 端点
2. `src/app/management/lists/[id]/page.tsx` - 新增批量上传 Modal 和逻辑

## 风险与约束

- 旧图删除为异步，不阻塞更新流程
- 批量上传过程无原子性保障，单张失败不回滚已成功的
