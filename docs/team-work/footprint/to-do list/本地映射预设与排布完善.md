# 本地映射预设与排布完善

## 实现内容

- 主文件夹已有记录时，除图片位置外，还要保存本次使用的预设类型与参数。
- 主文件夹已有记录且本次有新增文件时，新增文件默认按该记录中的预设类型继续补入，不再按弹窗默认值重新判断。
- 某地点图片组已有图片且本次有新增文件时：
  - 若原组周围仍有可拓展位置，则按原预设方向继续拓展。
  - 若原组周围已被其他组挤满，允许在本组范围内覆盖式补排，不强制向外继续扩张。
- 整齐排列与错位排列要适配任意比例图片，不再只按固定方格感知。
- 随机排列的随机附加距离从 `1-100` 改为 `10-60`。
- 需要新增一套基于标准区域与区域角度的图片组出现位置规则，用于确定新组落位与已有组增量扩张位置。
- 图片组数量小于 `5` 时，按标准区域直接占位。
- 图片组数量大于等于 `5` 时，按区域内图片组数量与图片数量的比例计算该区域可用总角度。
- 连线终点改为图片组矩形外围靠近地图的一点，不再使用图片组中心点。

## 实现方式

- 在本地映射记录里增加预设信息存储，至少保存：
  - `mode`
  - `gapX`
  - `gapY`
  - `staggerAxis`
- 读取旧记录时，若检测到该主文件夹已有预设记录，则新增文件自动复用该预设进行补排。
- 地点组增量补图时，先判断当前组外沿可用空间：
  - 可用时沿当前组朝外方向继续扩展
  - 不可用时改为在当前组包围盒内重新分配新增图位置，允许与组内旧排布做局部覆盖，但不要突破该组范围
- 整齐排列与错位排列的偏移计算改为基于每张图真实宽高推进，横向按真实宽度累计，纵向按真实高度累计。
- 随机排列保留现有规则，但随机附加值统一改为 `10-60`。
- 新组初次落位与区域扩张统一使用标准区域：
  - `N`：地图中心点正右上到正左上之间
  - `W`：地图中心点正左上到正左下之间
  - `S`：地图中心点正左下到正右下之间
  - `E`：地图中心点正右下到正右上之间
- 图片组数量小于 `5` 时：
  - 每个地点先按就近原则进入最接近的标准区域
  - 同一区域只允许一个主占用组
  - 若多个地点接近同一区域，则更靠该区域极值方向的地点优先占用
  - 其余地点改派到其他未占用区域
  - 双极值点按就近原则进入更接近的标准区域
- 图片组数量大于等于 `5` 时：
  - 每个地点先归属到某个标准区域
  - `N/S` 区内按经度从左到右排序
  - `W/E` 区内按纬度从上到下排序
  - 统计每个区域：
    - 图片组数量
    - 图片数量
  - 使用比例算法直接计算区域总角度，不再使用重叠带概念，也不在区域内再次做角度比例分配
  - 区域角度最小 `60°`，最大 `180°`
  - 比例算法：
    - `regionWeight = 0.65 * (regionGroupCount / totalGroupCount) + 0.35 * (regionPhotoCount / totalPhotoCount)`
    - `normalizedRegionWeight = regionWeight / maxRegionWeight`
    - `regionAngle = 60 + 120 * normalizedRegionWeight`
  - 区域中心角固定：
    - `E = 0°`
    - `S = 90°`
    - `W = 180°`
    - `N = 270°`
  - 区域实际角度范围：
    - `startAngle = centerAngle - regionAngle / 2`
    - `endAngle = centerAngle + regionAngle / 2`
- 区域内不再单独做角度分配，只保留顺序平铺：
  - 图片组按排序顺序，在所属区域总角度范围内顺序使用角度
  - 默认先平铺
  - 仅当图片组矩形发生重叠时，才启用半径错位
- 半径错位规则：
  - 半径错位只在顺序平铺位置不足且矩形重叠时启用
  - 大图片组在外，小图片组在里
  - 图片组大小按图片组矩形包围盒面积判断
- 连线规则：
  - 图片压线无所谓
  - 连线终点统一改为图片组矩形外围靠近地图的一点
  - 该点使用“地图中心点指向图片组矩形中心的向量”与图片组矩形边界的交点
