# 正式页排布算法迁移收敛-2026-06-01

- `src/app/(shell)/user/footprints/page.tsx`
- `src/components/OuterFrame.tsx`

- `src/components/LineCanvas.tsx`

- `docs/team-work/footprint/status.md`

## 改动目标
将测试页已验证的大组排布算法按约束迁入正式页，同时保留正式页既有的小组分区逻辑、图片组几何语义、文字锚点和交互体验，并完成默认最大视图与最小缩放的收敛。

## 本轮确认后的约束
- 图片组数量小于 `5` 时，继续使用正式页原有分区排布，不并入测试页新引擎。
- 图片组数量大于等于 `5` 时，自动排布主流程迁入测试页最终版算法。
- 图片组渲染几何继续以正式页当前图片组隐式矩形为准。
- 图片组文字锚点继续以正式页当前规则为准。
- 图片组连线端点最终继续使用正式页共享几何中的 `lineAnchorX / lineAnchorY`，不再使用测试页射线打到图片矩形的临时方案。
- 排布碰撞与收敛过程按测试页矩形语义执行。
- 默认最大视图按图片组隐式矩形四极值决定，且整体位置按真实四极值中心落位，不再以地图中心假定为基准。
- 最小缩放改为“预设最大视图的计算结果”，不再保留固定 `20%` 下限。

## 改动内容
- 正式页大组排布接入测试页当前收敛后的主算法：
  - 直线出射
  - 末端矩形避碰
  - 长度回收
  - 单线微调
  - 双线联动微调
  - 邻位最小间隔微调
- 小于 `5` 组时继续保留正式页原四向区域直占逻辑，避免小样本场景被测试页大组引擎带偏。
- 图片组碰撞语义与显示语义分离：
  - 排布避碰按测试页矩形参与收敛
  - 实际显示、标签与整体几何仍统一由正式页 `buildGroupGeometry()` 提供
- 连线端点恢复为正式页共享几何端点：
  - 统一使用 `buildGroupGeometry()` 生成的 `lineAnchorX / lineAnchorY`
  - 不再使用迁移中途引入的测试页射线交点端点
- 默认最大视图改为正式页图片组隐式矩形四极值拟合：
  - 以固定几何尺度计算每个组的 `overallRect`
  - 取所有组的最左、最右、最上、最下极值
  - 再加固定留白计算初始 fit 结果
  - 初始视图不再依赖当前缩放态回推
- 缩放交互收敛：
  - 自动 fit 只在 `fitViewKey` 变化时执行
  - 移除旧的 `50%` 门槛和固定 `20%` 下限
  - 最小缩放改为当前默认最大视图的 `scale`
- 性能回退修正：
  - POI 坐标换算由每帧循环改回地图事件驱动
  - 避免默认 fit 随缩放重复触发导致滚轮缩放发黏
- 右下角新增缩放比例浮层，便于肉眼确认默认最大视图与滚轮缩放结果。

## 当前结果
- 正式页与测试页当前职责已重新切开：
  - 测试页用于验证大组自动排布引擎
  - 正式页保留正式展示与交互语义，只迁入大组排布主流程
- 默认最大视图当前已不再受固定 `20%` 下限影响。
- 用户向外缩小时，最小只能缩到预设最大视图结果，不能再继续缩小。
- 文字锚点、连线端点、图片组实际显示矩形已重新收敛为正式页语义。
