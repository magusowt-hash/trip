# 普通地图足迹跨组数据库交接

## 目标
将普通地图来源地点从“同一用户同一 POI 只能存在一条记录”调整为“同一用户同一 POI 可存在于多个足迹组，但同组内不可重复”，以修复“添加到其他组”时报 500 的问题。

## 必须执行的数据库操作
- 执行 migration：
  - [drizzle/0014_user_map_footprints_group_scope.sql](/Users/apple/Desktop/codex/trip/drizzle/0014_user_map_footprints_group_scope.sql)

SQL 内容：

```sql
ALTER TABLE `user_map_footprints`
  DROP INDEX `user_map_footprints_user_poi_unique`,
  ADD UNIQUE INDEX `user_map_footprints_user_group_poi_unique` (`user_id`, `group_id`, `poi_id`);
```

## 操作原因
- 旧唯一索引是 `(user_id, poi_id)`
- 普通地图默认“已去”写入默认组后，再添加到其他组会再次插入相同 `poi_id`
- 旧索引会阻止第二条记录写入，导致接口 500

## 执行后预期
- 同一用户可将同一普通地图地点加入多个足迹组
- 同一组内仍不能重复加入同一普通地图地点
- 普通地图页“已去”仍保持用户维度全局状态

## 交接验证项
- 在普通地图页标记某地点为“已去”
- 进入足迹页，将该地点“添加到”另一个非默认组
- 预期：
  - 不再报 500
  - 目标组能看到该地点
  - 返回默认组后原地点仍存在
  - 普通地图页该地点仍显示为“已去”

## 风险说明
- 如果数据库尚未执行该 migration，仓库代码即使已更新，线上/测试环境仍会继续报 500
- 如果历史环境中唯一索引名称被手工改过，执行前需先 `SHOW INDEX FROM user_map_footprints;` 确认真实索引名
