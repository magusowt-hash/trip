# 云端同步与存储

## storage_files 本地足迹图片存储表

用于存储足迹页上传图片与本地映射图片的位置及展示尺寸元信息。当前 `place_title` 实际承载的是地点作用域键，而非纯地点名。

| 列名 | 类型 | 可空 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | SERIAL | NO | 自增 | 主键 |
| user_id | INT | NO | - | 用户 ID |
| place_title | VARCHAR(255) | NO | - | 地点作用域键，如 `fpgi_{footprint_group_items.id}` |
| filename | VARCHAR(500) | NO | - | 文件名 |
| size | BIGINT | NO | 0 | 文件大小 |
| pixel_width | INT | YES | NULL | 原始图片像素宽度 |
| pixel_height | INT | YES | NULL | 原始图片像素高度 |
| frame_x | DOUBLE | YES | NULL | 外框逻辑坐标 X |
| frame_y | DOUBLE | YES | NULL | 外框逻辑坐标 Y |
| created_at | TIMESTAMP | NO | NOW() | 创建时间 |

**索引：**
- `sf_user_file_unique`（UNIQUE）：`user_id, place_title, filename`
- `sf_user_place_idx`：`user_id, place_title`

**说明：**
- `pixel_width / pixel_height` 用于前端足迹页按真实宽高比参与预设排布
- 若历史数据为空，前端会回退到默认占位尺寸
- 本轮新增迁移：`drizzle/0013_storage_files_pixel_size.sql`

## local_map_roots 本地映射主文件夹记录表

用于保存用户在足迹页映射本地主文件夹后的正式记录，不再使用测试期 JSON 文本。

| 列名 | 类型 | 可空 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | SERIAL | NO | 自增 | 主键 |
| user_id | INT | NO | - | 用户 ID |
| root_name | VARCHAR(255) | NO | - | 主文件夹名称 |
| layout_mode | VARCHAR(32) | YES | NULL | 最近一次保存的预设类型：`grid / staggered / random` |
| layout_gap_x | INT | YES | NULL | 最近一次保存的横向间距参数 |
| layout_gap_y | INT | YES | NULL | 最近一次保存的纵向间距参数 |
| layout_stagger_axis | VARCHAR(32) | YES | NULL | 最近一次保存的错位方向：`horizontal / vertical` |
| created_at | TIMESTAMP | NO | NOW() | 创建时间 |
| updated_at | TIMESTAMP | NO | NOW() | 更新时间 |

**索引：**
- `local_map_roots_user_root_unique`（UNIQUE）：`user_id, root_name`
- `local_map_roots_user_id_idx`：`user_id`

**说明：**
- 同一用户同名主文件夹只保留一条正式记录
- 该表替代测试阶段 `test/footprint-local-map/local-map-store.json`
- 当前会把本地主文件夹最近一次使用的图片预设参数一并持久化，供后续新增文件自动补排复用

## local_map_assets 本地映射资源记录表

用于保存主文件夹下每张本地映射图片与足迹项的绑定关系及位置记录。

| 列名 | 类型 | 可空 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | SERIAL | NO | 自增 | 主键 |
| user_id | INT | NO | - | 用户 ID |
| root_id | INT | NO | - | 主文件夹记录 ID（关联 local_map_roots.id） |
| footprint_item_id | INT | NO | - | 绑定足迹项 ID |
| relative_path | VARCHAR(500) | NO | - | 相对主文件夹路径 |
| folder_name | VARCHAR(255) | NO | - | 所属第一层文件夹名 |
| name | VARCHAR(255) | NO | - | 文件名 |
| size | INT | NO | 0 | 文件大小 |
| last_modified | BIGINT | NO | 0 | 文件最后修改时间戳 |
| frame_x | DOUBLE | YES | NULL | 外框逻辑坐标 X |
| frame_y | DOUBLE | YES | NULL | 外框逻辑坐标 Y |
| pixel_width | INT | YES | NULL | 原始像素宽度 |
| pixel_height | INT | YES | NULL | 原始像素高度 |
| created_at | TIMESTAMP | NO | NOW() | 创建时间 |
| updated_at | TIMESTAMP | NO | NOW() | 更新时间 |

**索引：**
- `local_map_assets_root_path_unique`（UNIQUE）：`root_id, relative_path`
- `local_map_assets_user_item_idx`：`user_id, footprint_item_id`
- `local_map_assets_root_id_idx`：`root_id`

**说明：**
- 资源记录直接与足迹项绑定，不再只依赖地点名称
- 同一主文件夹下同一路径只保留一条记录
- 本轮新增迁移：`drizzle/0015_local_map_records.sql`

## cloud_mounts 云端挂载表

补充约定：足迹相册本地上传的 `storage_files.place_title` 在正式方案中不再表示纯地点名，而是表示地点作用域键。当前约定为 `fpgi_{footprint_group_items.id}`，用于隔离不同足迹组下的同名地点相册。

| 列名 | 类型 | 可空 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | SERIAL | NO | 自增 | 主键 |
| user_id | INT | NO | - | 用户 ID（关联 users.id） |
| footprint_item_id | INT | NO | - | 关联足迹项 ID（关联 footprint_group_items.id） |
| provider | VARCHAR(32) | NO | 'alist' | 云端提供商 |
| root_path | VARCHAR(500) | NO | - | 云端根路径 |
| status | VARCHAR(32) | NO | 'mounted' | 挂载状态：mounted, unmounted |
| connection_status | VARCHAR(32) | NO | 'unknown' | 连接状态：connected, disconnected, unknown |
| last_connected_at | TIMESTAMP | YES | NULL | 最后连接时间 |
| created_at | TIMESTAMP | NO | NOW() | 创建时间 |
| updated_at | TIMESTAMP | NO | NOW() | 更新时间 |

**索引：**
- `cloud_mounts_user_item_idx`：user_id, footprint_item_id
- `cloud_mounts_user_unique`（UNIQUE）：user_id
- `cloud_mounts_status_idx`：status, connection_status

**说明：** 用户将云端存储（如 Alist）挂载到某个足迹项，用于同步照片。
