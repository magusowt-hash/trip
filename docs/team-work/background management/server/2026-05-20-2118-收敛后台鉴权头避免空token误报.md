# 2026-05-20 21:18 收敛后台鉴权头避免空 token 误报

## 本次目标

- 修复后台部分页面在 token 为空时仍发送 `Authorization: Bearer null` 或 `Bearer undefined` 的问题
- 降低后台接口误报 `invalid token` 的概率，优先让 cookie 鉴权生效

## 实际改动

- 在 `src/app/management/admin-auth.tsx` 新增 `buildAdminHeaders`
- 在 `src/app/management/AdminTable.tsx` 统一使用 `buildAdminHeaders`
- 调整 `src/app/management/page.tsx`，看板请求不再直接读取 `localStorage`
- 调整 `src/app/management/posts/page.tsx`，帖子操作不再直接拼接本地 token
- 调整 `src/app/management/users/[id]/page.tsx`，详情页与足迹地图改为统一鉴权头构造

## 结果

- 未登录或本地 token 丢失时，请求将不再主动发送空 Bearer 头
- 后台管理接口可优先依赖登录 cookie，减少 `invalid token` 噪音

## 注意

- 该修复不能解决 `ECONNREFUSED 127.0.0.1:3306`
- 数据库拒绝连接仍需在服务器侧恢复 MySQL 监听与应用连接
