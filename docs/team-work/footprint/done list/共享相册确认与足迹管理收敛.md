# 共享相册确认与足迹管理收敛

## 改动范围
- `src/components/FootprintGroupPanel.tsx`
- `src/app/(shell)/user/footprints/page.tsx`
- `src/app/api/footprints/groups/[id]/items/route.ts`
- `src/services/storage.ts`

## 改动目标
收敛足迹组管理入口，并修复“添加到某组”时共享相册确认链路的显示与数据一致性问题。

## 改动思路
将新建组入口统一收敛到足迹管理弹窗，避免菜单侧双入口；同时拆开“是否共享相册”的前端显示问题与后端相册作用域继承问题，确保“是/否”两条分支都与实际相册数据一致。

## 改动内容
- 删除足迹菜单中的“新建分类”按钮，仅保留足迹管理弹窗中的“新建足迹组”入口。
- 将共享相册确认弹窗改为 portal 渲染，保证其层级始终高于管理弹窗。
- 修复“添加到某组”时的相册探测逻辑：
  - 补查同地点其他足迹项的相册作用域
  - 兼容旧地点名作用域
- 修复“选择否仍继承相册图片”的问题：
  - 非共享时不再写入共享 `albumScopeKey`
  - 旧兼容迁移逻辑只迁移当前足迹项，不再按同名地点自动扩散
