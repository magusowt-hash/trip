# 开发任务拆分清单

## 目标

- 将当前已完成的 Footprint 网盘挂载方案拆成可直接执行的开发任务
- 按后端、前端、联调、验证分组
- 便于多人并行推进，同时避免改动边界混乱

## 当前方案边界

第一阶段范围：

- 基于当前 Footprint 数据库收敛实现
- 正式展示继续只走 `storage_files`
- 复用 `footprint_group_items.cloud_folder`
- 新增 `cloud_mounts`
- 新增 `cloud_sync_logs`
- 按需新增轻量 `cloud_assets`
- 所有挂载入口放在足迹项详情菜单的 `挂载网盘`
- 交互主入口为弹窗

## 任务分组

- A. 数据库与后端基础
- B. 同步与匹配服务
- C. 前端菜单与弹窗
- D. 页面展示与待匹配提示
- E. 联调与验证

## A. 数据库与后端基础

### A1. 扩展 `storage_files`

任务内容：

- 为 `storage_files` 增加：
  - `source_type`
  - `source_ref`
  - `source_folder`
- 增加：
  - `sf_user_source_idx`
  - `sf_user_source_ref_unique`

产出：

- drizzle migration
- `src/db/schema.storage.ts` 更新

依赖：

- 无

风险：

- 旧唯一键与云端写入冲突

### A2. 新建 `cloud_mounts`

任务内容：

- 创建挂载状态表
- 在 schema 中导出

产出：

- drizzle migration
- `src/db/schema.cloud.ts`
- `src/db/schema.ts` 导出

依赖：

- 无

### A3. 新建 `cloud_sync_logs`

任务内容：

- 创建同步日志表
- 在 schema 中导出

产出：

- drizzle migration
- `src/db/schema.cloud.ts`

依赖：

- A2

### A4. 条件新建 `cloud_assets`

任务内容：

- 创建未匹配资源轻量表

产出：

- drizzle migration
- `src/db/schema.cloud.ts`

依赖：

- A2

说明：

- 若决定未匹配只记目录摘要，可延后

## B. 同步与匹配服务

### B1. 挂载状态查询接口

任务内容：

- 实现：
  - `GET /api/footprints/cloud/mount/status?itemId=...`
- 聚合：
  - `cloud_mounts`
  - 最新 `cloud_sync_logs`
  - 未匹配统计

产出：

- route handler
- service 层状态聚合函数

依赖：

- A2
- A3
- A4（如果启用未匹配资源持久化）

### B2. 挂载网盘接口

任务内容：

- 实现：
  - `POST /api/footprints/cloud/mount/connect`
- 为足迹项建立挂载关系
- 校验：
  - 用户权限
  - 足迹项存在
  - 挂载冲突

产出：

- route handler
- mount service

依赖：

- A2

### B3. 云端连接检测

任务内容：

- 基于现有 AList 接入能力做连接测试
- 更新 `cloud_mounts.status`

产出：

- service function

依赖：

- A2
- 现有 `src/services/alist.ts`

### B4. 手动同步接口

任务内容：

- 实现：
  - `POST /api/footprints/cloud/sync`
- 调用：
  - 目录扫描
  - 地点匹配
  - 已匹配写 `storage_files`
  - 未匹配写 `cloud_assets`
  - 同步日志写入

产出：

- route handler
- sync service

依赖：

- A1
- A2
- A3
- A4
- B3

### B5. 已匹配云端图片视图接口

任务内容：

- 实现：
  - `GET /api/footprints/cloud/outer-frame?itemId=...`
- 从 `storage_files` 中返回当前足迹项可展示的云端图片

产出：

- route handler
- DTO 组装函数

依赖：

- A1
- B4

### B6. 未匹配提示接口

任务内容：

- 实现：
  - `GET /api/footprints/cloud/hints?itemId=...`
- 按目录聚合返回：
  - `folderName`
  - `assetCount`

产出：

- route handler
- hint service

依赖：

- A4
- B4
