# 挂载网盘弹窗交互草案

## 目标

- 为 Footprint 足迹项的“挂载网盘”菜单项定义明确的前台交互
- 让挂载、连接状态、手动同步、同步结果、未匹配提示都在一个弹窗内闭环
- 保持页面主体简洁，不把云端资源操作扩散到全局界面

## 已确认前提

- `挂载网盘` 位于足迹项详情菜单中
- 菜单项固定为：`设为默认 / 重命名 / 删除 / 挂载网盘`
- 每个挂载网盘仅支持一个足迹项
- 网盘根目录第一层必须是地点文件夹名
- 同步需要用户手动触发
- 已绑定云端图片进入正式 Footprint 后，与本地图片统一显示，不区分来源
- 未匹配目录不进入正式展示，只做提示

## 菜单状态规则

`挂载网盘` 菜单项状态：

- 未挂载：默认样式
- 已挂载未连接：红色
- 已挂载已连接：绿色

说明：

- 颜色只表达“该足迹项对应网盘的当前状态”
- 不在菜单项中附加额外长文案，避免菜单过重

## 弹窗定位

弹窗名称建议：

- `挂载网盘`

说明：

- 第一阶段重点是“当前足迹项关联的网盘空间”
- 不需要扩展成复杂资源管理器形态

## 弹窗默认首页

默认进入：

- `网盘状态总览`

说明：

- 用户一进入就能看到是否已挂载、是否可连接、最近同步情况、是否有未匹配目录
- 这比直接扔给用户一个同步按钮更稳

## 弹窗结构

建议分为 4 个区域：

1. 状态区
2. 操作区
3. 同步结果区
4. 未匹配区

## 1. 状态区

### 展示内容

- 当前足迹项名称
- 网盘挂载状态
- 网盘连接状态
- 最近同步时间
- 最近同步结果摘要

### 状态枚举

#### 未挂载

展示：

- 状态：`未挂载`
- 说明：`该足迹项尚未关联网盘`

#### 已挂载未连接

展示：

- 状态：`已挂载，连接异常`
- 说明：`已保存网盘配置，但当前无法访问`

#### 已挂载已连接

展示：

- 状态：`已挂载，连接正常`
- 说明：`可手动同步网盘图片`

### 推荐视觉

- 状态标签使用明确颜色：
  - 灰：未挂载
  - 红：连接异常
  - 绿：连接正常

## 2. 操作区

### 未挂载状态

按钮建议：

- `选择网盘`
- `确认挂载`

说明：

- 第一阶段如果网盘来源是用户已有挂载空间，则这里重点是“把当前足迹项绑定到一个网盘”
- 不在这里展开复杂目录浏览器

### 已挂载未连接状态

按钮建议：

- `重试连接`
- `重新选择网盘`

说明：

- 用户不应直接先点同步
- 先解决连接状态

### 已挂载已连接状态

按钮建议：

- `同步网盘`
- `更换网盘`
- `解除挂载`

说明：

- `同步网盘` 是主操作
- `更换网盘` 和 `解除挂载` 是次操作

## 3. 同步结果区

### 触发方式

用户点击：

- `同步网盘`

### 同步中状态

展示：

- `正在同步网盘...`
- 禁用重复点击

### 同步完成后展示

结果面板内明确展示：

- 导入图片数
- 成功匹配目录数
- 未匹配目录数
- 跳过图片数
- 最近同步时间

### 文案方向

示例：

- `本次同步导入 184 张图片，匹配 8 个地点目录，2 个目录未匹配`

### 同步失败展示

展示：

- 错误摘要
- `重试同步` 按钮

示例：

- `同步失败，当前无法访问网盘，请稍后重试`

## 4. 未匹配区

### 展示原则

- 未匹配信息放在弹窗内长期可见
- 不只在同步结束瞬间出现

### 展示内容

每条未匹配目录展示：

- 目录名
- 图片数

示例：

- `未识别地点A · 11 张`
- `2025夏季素材 · 6 张`

### 第一阶段不展示

- 单张图片列表
- 缩略图墙
- 自动候选地点列表

原因：

- 第一阶段先控制信息密度
- 用户先知道“哪个目录没匹配上”就够了

### 空状态

若没有未匹配目录：

- `当前没有未匹配目录`

## 弹窗状态流转

### 场景 1：首次进入，未挂载

```text
打开弹窗
  -> 状态区显示“未挂载”
  -> 操作区显示“选择网盘 / 确认挂载”
  -> 同步结果区为空
  -> 未匹配区为空
```

### 场景 2：已挂载但连接失败

```text
打开弹窗
  -> 状态区显示红色异常状态
  -> 操作区显示“重试连接 / 重新选择网盘”
  -> 同步按钮不可作为主按钮
```

### 场景 3：已挂载且连接正常，未同步

```text
打开弹窗
  -> 状态区显示绿色正常状态
  -> 操作区主按钮为“同步网盘”
  -> 同步结果区提示“尚未同步”
```

### 场景 4：同步成功且存在未匹配目录

```text
点击“同步网盘”
  -> 同步中
  -> 返回同步结果
  -> 同步结果区显示导入/匹配/未匹配摘要
  -> 未匹配区列出目录名 + 图片数
```

### 场景 5：同步成功且全部命中

```text
点击“同步网盘”
  -> 同步结果区显示成功摘要
  -> 未匹配区显示“当前没有未匹配目录”
```

## 页面常驻入口配合

虽然弹窗内会显示未匹配区，但页面主体仍建议保留常驻“待匹配”入口。

作用：

- 用户关闭弹窗后，仍知道还有待处理目录
- 与弹窗形成轻重配合

推荐展示：

- `待匹配 2`

说明：

- 页面常驻入口只做提醒和进入弹窗/处理页跳转
- 不承载完整处理逻辑

## 数据依赖

弹窗至少需要以下接口数据：

- 当前足迹项的挂载状态
- 当前足迹项的连接状态
- 最近一次同步摘要
- 当前未匹配目录列表

建议接口：

- `GET /api/footprints/cloud/mount/status?itemId=xxx`
- `POST /api/footprints/cloud/mount/connect`
- `POST /api/footprints/cloud/sync`
- `GET /api/footprints/cloud/hints`

说明：

- 第一阶段可让 `sync` 与 `hints` 仍然按当前足迹项上下文工作

## 前台优化建议

### 建议 1

同步结果区保留最近一次结果，而不是操作后立即消失。

### 建议 2

未匹配区默认展开，不要藏得太深。

### 建议 3

如果连接异常，不要让用户误以为可以直接同步。

### 建议 4

菜单颜色和弹窗状态颜色保持一致，减少理解成本。

## 第一阶段不做的内容

- 弹窗内直接修正目录绑定
- 弹窗内浏览全部图片缩略图
- 多网盘同时挂载到一个足迹项
- 自动后台同步
- 目录级拖拽映射

## 推荐实现顺序

1. 先做菜单状态显示
2. 再做弹窗状态总览
3. 再接“同步网盘”动作
4. 再接同步结果面板
5. 最后接未匹配目录区和页面常驻入口
