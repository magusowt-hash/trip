## 改动目标
完成榜单地点收藏主来源收敛，并让普通地图“已去”在用户足迹页、分享视图、后台查看链路中与榜单页“已去”保持统一体验。

## 改动思路
收藏与已去拆开收敛：
- 榜单地点收藏从 `users.favorite_lists` JSON 改为独立关系表，避免继续依赖用户表 JSON 主写。
- 榜单“已去”继续保留 `footprint_group_items`。
- 普通地图“已去”继续保留 `user_map_footprints`，但在默认足迹组读取、展示、相册作用域和后台查看层做统一兼容，让用户端体验尽量与榜单“已去”一致。

## 改动内容
- 新增 `user_list_favorites` 表，榜单页、地点收藏页、后台用户详情改为从该表读取地点收藏。
- 默认足迹组读取逻辑改为合并：
  - `footprint_group_items`
  - `user_map_footprints`
- 默认足迹组数量统计同步合并普通地图“已去”数量。
- 分享视图 `/api/footprints/view` 同步支持普通地图“已去”数据与图片读取。
- 普通地图页用户文案统一为“已去”，不再使用“加入足迹 / 已加入足迹”。
- 修复普通地图“已去”刷新后状态丢失问题：
  - 搜索结果返回后按 `amapPoiId` 或 `name + lng + lat` 回填已保存 `poiId`
- 新增普通地图“已去”相册作用域：
  - `mfpi_<poiId>`
- 普通地图“已去”在用户足迹页补齐以下能力：
  - 从当前已去组移除
  - 打开相册
  - 上传图片
  - 添加到其他已去组
- 后台足迹查看与用户详情页同步兼容普通地图“已去”的分组展开与照片读取。
- 修复默认足迹组 items 接口缺少 `sql` 导入导致普通地图“已去”不显示的问题。
