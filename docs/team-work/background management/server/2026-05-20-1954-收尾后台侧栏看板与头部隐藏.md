# 2026-05-20-1954 - 收尾后台侧栏看板与头部隐藏

## 改动范围

- `src/app/layout.tsx`
- `src/app/management/layout.tsx`
- `src/app/management/layout.module.css`
- `src/app/management/page.tsx`
- `src/app/management/page.module.css`

## 改动目标

本次要解决后台管理壳层交互仍不够完整的问题，包括前台全局 Header 误入后台、侧边栏不可独立滚动、分类不可收起，以及看板页信息展示形式仍不符合最终要求。

## 改动思路

继续围绕后台壳层做收尾优化，把后台首页的职责进一步收敛为“趋势 + 数据版”，同时让后台侧边栏成为真正独立可用的导航区，并彻底屏蔽前台 Header 对后台页的干扰。

## 改动内容

- 在根布局中将 `/management` 路径加入 Header 隐藏条件，后台页不再渲染前台站点头部。
- 为后台侧边栏增加独立滚动区，超出内容只在侧边栏内部滚动。
- 为 `系统管理`、`用户管理` 两个分类增加可收起交互，并根据当前页面默认展开对应分组。
- 删除看板页原有模块导航区，让入口职责完全回到左侧侧边栏。
- 将后台看板页重构为双列结构：
  - 左侧趋势图
  - 右侧数据版列表
- 将看板视觉风格收敛为纯白极简样式，移除原先较重的彩色统计卡表现。
